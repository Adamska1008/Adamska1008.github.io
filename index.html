Linux C中基于句柄的文件操作

发表于 2021-10-20 更新于 2021-10-24 分类于 操作系统

fcntl.h 与 unistd.h 定义了一系列基于句柄的文件操作函数以及常量。其读写文件基于句柄，与stdio.h中基于file的文件读写有很大的不同。考虑到Linux一切皆文件的特性，fcntl.h有着更好的泛用性，且更接近底层。

虽然这些操作更加接近底层，但却未必比直接使用库函数来的快速，因为从用户态切换到内核态需要额外的时间。在实际使用过程中，最好针对此点进行优化。

句柄（文件描述符）

句柄（Handle）是一个用于标识文件的标识符，其在C中为int类型。句柄在Linux中，不仅可以标识狭义的文件，还可以标识输入输出流或套接字。

标准输入流的句柄为0，标准输出流的句柄为1，标准错误流的句柄为2，对应的常量作为STDIN_FILENO, STDOUT_FILENO, STDERR_FILENO存储在fcntl.h中

write()

ssize_t write(int fildes, const void* buf, size_t nbytes)从缓冲区buf中向句柄指向的文件写入nbytes的数据。在一般的文件中，若nbytes大于文件本身的长度，则文件将被扩容。特别地，若句柄指向pipeline或一些具有限制的媒介中，数据量超过容量的部分将被延迟接收，直到获得充足的容量位置。

write返回中断前被接受的字节数。如果write在没有写入任何字节的情况下被中断，将返回-1，并将errno设置为EINTR。特别地，若nbytes == 0，则返回0，并不进行任何操作。

write写入文件的方式由open函数的参数决定。若为O_APPEND，则在文件末尾续写数据，若为O_TRUNC，则会覆盖文件本身的数据。特别地，对于pipeline，所有写入将直接附加到pipeline的末尾。

open()

要为一个句柄赋值，可以使用open打开文件，或者socket构建套接字。新的句柄从3开始计数，不同进程有着独立的句柄计数空间。

int open(const char *path, int oflag,...)表示打开path对应的文件，参数为oflag，返回这个文件持有的句柄。oflag的具体内容可以查阅open词条。open函数建立起文件与句柄之间的联系，open返回的句柄为可用句柄中的最小值，即从3,4,5..计数。当一个句柄被关闭后，其将被销毁，新生成的句柄将覆盖之前的句柄，例如：

句柄同样可以使用socket(),accept(),connect()等函数构建，这部分的内容会放在之后的文章中。

当句柄构建失败时，返回值为-1。

read()

同样的，可以使用read()函数进行读取操作

ssize_t read(int fildes, void *buf, size_t nbyte)从句柄指向的文件中读取nbytes的数据到buf中。

close()

在文件读写结束之后，使用int close(int fildes);关闭管道，防止资源泄露，同时，close一个句柄也意味着一切对文件的锁都将被解除。close一个pipeline时，其中的所有内容将被遗弃。返回0表示关闭成功，-1表示关闭操作被中断。

流式的read(), write()

read()与write()函数是基于流的读写操作，这意味着数据的收发是无边界的。这也意味着read()与write()是适用于TCP的。以下方的程序为例

可以看到，我们分两次向标准输出流调用了write()，而只需要调用一次read()就能收到之前发送的所有消息，这便是流式传输。

实际上，write()在调用之后并非立即传送数据，而是先将数据移送至缓冲区，read()同样也是直接向缓冲区读取数据。

read()的BUF_SIZE即为可从缓冲区读取数据的上限，虽然我们设置BUF_SIZE为1024，但并不一定能读取到1024字节的数据，因为这只是读取数据的上限。

调用一次read()时，读取到缓冲区为空或读取的数据量达到上限为止。

即使pipeline两端中一方关闭pipeline，缓冲区也不会消失，而是直到两端都关闭时才消失。

最大异或对

在给定的 $N$ 个整数 $A_1，A_2……A_N$ 中选出两个进行 $xor$（异或）运算，得到的结果最大是多少？

输入格式

第一行输入一个整数 $N$。

第二行输入 $N$ 个整数 $A_1 \sim A_N$。

输出格式

输出一个整数表示答案。

数据范围

$1≤N≤10_5,$
$0≤A_i<2_{31}$

输入样例

输出样例

题解

考虑对一个确定的 $a$，如何求使其异或值最大的 $b$ ？显然，$b$ 即为对 $a$ 按位取反。

假设有两个数 $b_i,b_j$，如何判断他们与 $a$ 异或的值哪个大？由大到小按位进行位运算即可。若在高位第k位上，$b_i[k] \ xor \ a[k] == 1，而 b_j[k] \ xor \ a[k] == 0$，那么必然有 $b_i \ xor \ a > b_j \ xor \ a$，这是因为比大小即按位判断。

对于一组数字，使用trie存储01串。对每一个数字，从高位到低位查找是否有异或和为1的节点，若有则从此开始找，否则只能下降到另一个节点。

加成序列

满足如下条件的序列 $X$（序列中元素被标号为 $1、2、3…m$）被称为“加成序列”：

  1. $X[1]=1$
  2. $X[m]=n$
  3. $X[1]<X[2]<…<X[m−1]<X[m]$
  4. 对于每个 $k（2≤k≤m）$都存在两个整数 $i$ 和 $j$ （$1≤i,j≤k−1$，$i$ 和 $j$ 可相等），使得 $X[k]=X[i]+X[j]$。
    你的任务是：给定一个整数 $n$，找出符合上述条件的长度 $m$ 最小的“加成序列”。

如果有多个满足要求的答案，只需要找出任意一个可行解。

输入格式

输入包含多组测试用例。

每组测试用例占据一行，包含一个整数 n。

当输入为单行的 0 时，表示输入结束。

输出格式

对于每个测试用例，输出一个满足需求的整数序列，数字之间用空格隔开。

每个输出占一行。

数据范围

$1≤n≤100$

输入样例

输出样例

题解

迭代加深算法经典题。

首先分析一下如何搜索。题目要求每一个数都必须是前两个数的和，那么我们可以枚举每两个数来生成后一个数。

然后分析一下如何剪枝，首先是如何枚举两个数的和。为了保证每一层的分支尽量小，我们需要先枚举较大的和。其次，枚举可能有重复，比如1 2 3 4, $2 + 3 = 1 + 4 = 5$，同一个数枚举了两次，所以需要开一个bool数组记录哪些数已经使用过了。

最后，我们分析一下这个题的特点。n的上界是100，很大，而且我们很容易构造出来一个数组1 2 3 4 ... 98 99 100，可见搜索过程中是有可能搜到100层的。但是，不难看出，题目中所需的最小数组长度其实是很小的，77才9层，所以答案层数明显小于上界层数，这个时候我们可以使用迭代加深，枚举深度上限来进行dfs，就能防止搜到过于巨大的状态导致TLE。

最大子序和

输入一个长度为 $n$ 的整数序列，从中找出一段长度不超过 $m$ 的连续子序列，使得子序列中所有数的和最大。

注意： 子序列的长度至少是 $1$。

输入格式

第一行输入两个整数 $n,m$。

第二行输入 $n$ 个数，代表长度为 $n$ 的整数序列。

同一行数之间用空格隔开。

$1≤n,m≤300000$

题解

$O(n^2)$ 的最大子序和有两种写法，一个是dp，一个是前缀和，一般认为dp比前缀和好，因为前缀和可能爆int甚至long long，但是前缀和可以使用单调队列优化为 $O(n)$。

以样例为例，前缀和为1 -2 3 4 2 5。对于前缀和，我们实际上只关心一段序列中最大值与最小值之差的最大值（必须保证最小值在最大值之前）。我们看到最小值为 -2, 最大值为 5 ,且 -2 在 5 之前，那么最大子序列一定是 7 （忽略长度限制）。

这显然就是一个单调队列，队首为一段区间内的最小值的序号，队尾为最大值的序号。
