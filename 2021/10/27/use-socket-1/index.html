使用socket进行简单的通信(1)

发表于 2021-10-27 更新于 2022-02-12 分类于 使用socket进行简单的通信

socket即套接字，本文不介绍基本的理论知识，仅简单说明如何在代码中使用socket。

socket()

int socket(int __domain, int __type, int __protocol)接受三个参数，返回操作套接字的句柄。三个参数分别为：

  • __domain:即通信域(communication domain)，指定了网络层的协议族，常用的如AF_INET，IPv4网络协议。
  • __type，为socket的类型，常用的有SOCK_STREAM，SOCKET_DGRAM，前者用于tcp，后者用于udp。
  • __protocol，传输层协议，即TCP,UDP等。当指定__type之后，传入形参0即可分配默认__protocol参数。

socket函数保存在<sys/socket>中。

获得socket的句柄后，我们可以使用此句柄进行write()，read() 等操作。

当我们创建了一个套接字后，这个套接字还未与一个地址连接，我们必须为套接字分配一个地址，才能够使用套接字进行通讯。为此，我们需要bind()函数。

sockaddr

在介绍bind()之前，需要先介绍sockaddr，sockaddr有两个成员变量：sin_family以及sa_data。sin_family的含义与上文中的__domain相同，同样指网络协议族。sa_data是一个14字节的char数组，用于描述这个地址，由于sin_family不同，sa_data的内容也不同。由于C不是面向对象的语言，不能通过继承体现不同sin_family下sockaddr实现的不同，所以我们通常不直接使用sockaddr，而是使用类似于sockaddr的其他结构体，在函数调用时作强制类型转换。例如，在internet环境下，使用sock_addr_in结构体。

sock_addr_in中，sa_data被替换为三个变量，分别为sin_addr，sin_port与sin_zero。sin_zero没有用处，仅用于字节对齐。sin_port为端口号，为int类型，sin_addr为ip地址，是一个结构体，其内部为s_addr，是int类型，保存了ip地址的信息。

网络字节序

sin_port与sin_addr.s_addr均为int类型。当在网络中传输数据时，规定数据以小端序进行传输。然而，许多操作系统使用大端序保存数据。

一般来说，在操作系统层面，会在网络通信时自动改变字节序。然而，对于sockaddr来说，我们需要手动设置。使用 uint16_t htonl(uint16_t) 或 uint32_t htons(uint32_t)函数，即可完成转化。前者用于32位整型，后者用于16位整型。hton表示host to net。这两个函数保存在<arpa/inet>中。

除此之外，我们还会经常使用到int inet_addr(const char*)函数。这个函数的作用是把点分割字符串表示的ip地址转化为实际的ip地址。我们书写ip时，常常使用点分割，比如127.0.0.1，但实际上的ip地址是32位字节码。虽然编写一个转换函数并不难，但标准库提供了一个转换函数inet_addr(const char*)。同时，这个函数默认输出小端序的ip地址，所以不需要另外使用htonl。

可以看到字符串被转换成了二进制ip，且确实是以小端序保存的。

此函数同样可以用于判断ip地址是否合法。若不合法，返回-1。

bind()

int bind(int __fd, const struct sockaddr *__addr, socklen_t __len)传入三个参数，若返回值为-1，则表明bind()失败。三个参数分别为socket句柄，socket将被分配的地址以及其大小。

当使用bind()之后，即对socket分配地址，socket可以进行通信了。
