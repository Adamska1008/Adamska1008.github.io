指令系统

指令概述及格式

基本概念：

  • 计算机语言中的基本单词称为指令，一台计算机的全部指令称为指令集（instruction set）。
  • 指令字长：指令中包含的二进制位数
    • 多字长指令：解决寻址较大存储空间的问题，多次取指影响速度
    • 等长指令：指令长度固定

指令分类：

  • 根据计算机层次结构：
    • 高级语言指令
    • 汇编语言指令
    • 机器语言指令
  • 根据指令中地址码字段个数分类
    • 三地址指令：[OP|A1|A2|A3] →(A1)OP(A2)=(A3)
    • 二地址指令：[OP|A1|A2] →(A1)OP(A2)=(A1)
    • 一地址指令：[OP|A1] →(AC)OP(A1)=(A1)
    • 零地址指令：[OP]，如停机、空操作、堆栈指令
  • 根据指令中操作数的物理位置分类：[OP|A1|A2]
    • 存储器-存储器（SS）型
    • 寄存器-寄存器（RR）型
    • 寄存器-存储器（RS）型
  • 根据指令的功能分类：
    • 传送指令：MOV、PUSH/POP、IN/OUT等
    • 定点算术运算指令：ADD、SUB、INC、CMP、MUL等
    • 位运算指令：NOT、AND、OR、SHL、SAL等
    • 控制转移指令：JMP、JNE、CALL、RET

指令格式：

  • 操作码字段：指令完成的功能
    • 定长操作码位数与最大指令数量有关：Lop=⌈ log2n ⌉
    • 变长操作码向不用的地址码字段扩展
  • 地址码字段：
    • 作用和影响与寻址方式和位数有关

寻址方式

指令寻址：
使用程序计数器（program counter） 保存当前运行的指令地址。每执行一条指令，PC加1。控制转移指令可能修改PC实现跳跃寻址。

数据寻址：
记地址E处的数据为S=(E)，操作数为D。

  • 立即数寻址：地址码字段是数据本身
    • 例：MOV AX, 200H
    • 特点：
      • 数据与指令一并读入寄存器，速度快
      • 便于程序设计
      • 数据大小受字段位控制
  • 寄存器寻址：操作数在寄存器内部
    • 例：MOV AX, BX
    • 特点：
      • 操作数在寄存器中，速度快
      • 能访问的数据大小与计算机字长有关
      • 地址字段位数与计算机通用寄存器数量相关：⌈ log2n ⌉
  • 直接寻址：地址码字段为操作数在内存中的地址
    • E=D, S=(D)
    • 例：MOV AX, [200H]
    • 特点：
      • 需要访问主存，速度慢
      • 地址字段位数决定了访问空间大小
  • 间接寻址：地址码是操作数主存地址的地址
    • E=D, S=((D))
    • 例：MOV AX, I[200H]
    • 特点：
      • 解决了直接寻址下地址字段位数限制访问范围大小的问题
      • 访问主存两次，太慢
  • 寄存器间接寻址：地址码是寄存器编号
    • E=®，S=(®)
    • 例：MOV AX, [BX]
    • 特点：
      • 解决了直接寻址下地址字段位数限制访问范围大小的问题
      • 访问主存一次
  • 相对寻址：以PC为基地址，地址码为偏移量，相加得到地址
    • E=D+(PC)
    • 例：JMP 200
    • 特点：
      • 节省地址位数
      • 由于PC的值在取指阶段完成修改，E需要额外增加
  • 基址寻址：指定一个基址寄存器B
    • E=D+(B), S=(E)
    • 例：MOV AX 32[B]
    • 特点：
      • 使用基址寻址可以访问更大空间
      • 对某一程序，基址设定后不变，通过修改D访问数据
      • 访问一次内存
  • 变址寻址：指定一个变址寄存器X，内容可以改变
    • E=D+(X)，S=(E)
    • 例：MOV AX, 32[SI]
    • 特点：
      • 不改变指令即可改变数据的有效地址，常在循环中使用

某计算机采用双字节长指令，内存基于字节寻址，指令数据用补码表示，PC的值在取指阶段完成修改

  1. 若采用相对寻址，当前地址为2003H，要求转移后目标地址为200AH，则该指令地址码为多少。
  2. 若采用相对寻址，当前地址为2003H，要求转移后目标地址为200AH，则该指令地址码为多少。
  1. 200AH-(2003H+2)=5H
  2. 2001H-(2008H+2)=-9=F7H

指令格式设计

主要内容：

  • 根据指令数量以及是否支持操作码扩展，确定操作码字段的位数
  • 根据对操作数的要求确定地址码字段的个数
  • 根据寻址方式，确定寻址方式字段
  • 确定采用定长还是变长指令

字长16位，主存64K，指令单字长单地址，80条指令，寻址方式有直接、间接、相对、变址

80条指令 → OP字段需要7位
4种寻址方式 → 寻址方式特征位2位
单地址位长度：16-7-2=7位
指令格式：[OP|X|A]

MIPS指令集

MIPS是精简指令集RISC（Reduced Instruction Set Computer），有以下特点：

  • 简单的Load/Store结构
  • 易于流水线CPU设计
  • 易于编译器开发
  • 寻址方式、指令操作简单

指令格式

  • R型指令：Rs、Rt为源操作数，Rd为目标操作数
  • I型指令：
    • Load/Store：Rs和立即数为源操作数，Rt为目标操作数
    • 条件转移：Rs、Rt均为源操作数
  • J型指令：26位立即数作为跳转目标地址的部分地址

MIPS寄存器

寄存器名 寄存器编号 用途说明
$s0 0 保存固定常数0
$at 1 汇编器临时变量
$v0~$v1 2~3 子函数调用返回结果
$a0~$a3 4~7 函数调用参数1~3
$t0~$t7 8~15 临时变量，函数调用时不需要保存和恢复
$s0~$s7 16~23 函数调用时需要保存和恢复的寄存器变量
$t8~$t9 24~25 临时变量，函数调用时不需要保存和恢复
$k0~$k1 26~27 中断、异常处理程序使用
$gp 28 全局指针变量（Global Pointer）
$sp 29 栈指针变量（Stack Pointer）
$fp 30 帧指针变量（Frame Pointer）
$ra 31 返回地址（Return Address）

MIPS寻址

MIPS指令集不单设置寻址方式字段。

  • R型指令：由op和func字段共同隐含说明当前的寻址方式

  • I型和J型指令：由op字段隐含说明当前的寻址方式

  • 立即数寻址：

    • I型指令：[OP|Rs|Rt|立即数]
    • 举例：addi s1, s2, 10 → $s1=$s2+10

  • 寄存器直接寻址：

    • R型指令：[OP|Rs|Rt|Rd|shamt|funct]
    • 举例：add $t0, $s1, $s2 → $t0=$s1+$s2

  • 基址寻址：

    • I型指令：[OP|Rs|Rt|立即数]
    • 立即数带符号扩展为32位
    • $Rs为基址寄存器，($Rs)+立即数=E, S=(E)
    • 使用基址寻址的指令：lw，sw，lh，sh，lb，lbu等

  • 相对寻址

    • I型指令（Rs、Rt用于条件判断）
    • 立即数带符号扩展成32位后左移2位
    • E=(PC)+立即数

  • 伪直接寻址（页面寻址）：

    • J型指令
    • 立即数左移2位
    • E=PC作为高四位+立即数

详细指令

R型指令
[OP|Rs|Rt|Rd|shamt|funct]

  • OP：操作码，所有R型指令OP全为零
  • Rs：寄存器编号，对应第一个操作数
  • Rt：寄存器编号，对应第二个操作数
  • Rd：寄存器编号，对应保存结果
  • shamt：常数，在移位指令中使用
  • funct：功能码，指定指令具体功能

三类指令：

  • 三寄存器R型指令：
    • add、sub、and、or、xor
    • 指令功能：$Rd ← $Rs op $Rt
  • 二寄存器R型指令
    • sll、srl、sra
    • $Rs=0、shamt=sa
    • 指令功能：$Rd ← &Rt shift sa
  • 一寄存器R型指令
    • jr
    • 只有$Rs非0
    • PC ← $Rs

I型指令
[OP|Rs|Rt|立即数]

  • OP：操作码
  • Rs：源寄存器
  • Rt：目的寄存器
  • Imm：立即数，第二个源操作数

四类指令：

  • 面向运算：
    • addi、andi、ori、xori
    • $Rt ← $Rs OP E(Imm)
    • addi使用符号扩展，其他指令使用0扩展
  • 面向访存
    • lw
      • lw Rt, Imm(Rs)
      • $Rt ← mem[$Rs+E(Imm)]
    • sw
      • sw Rt, Imm(Rs)
      • mem[$Rs+E(Imm)] ← $Rt
  • 面向数位设置
    • lui
      • lui Rt, Imm
      • $Rt ← Imm << 16 （空位补零）
  • 面向条件转移分支
    • beq（$Rs == $Rt）、bne（$Rs != $Rt）
    • if (cond) PC ← PC+E(Imm)<<2
    • 标准的PC相对寻址
    • Imm先符号扩展成32位，再左移两位

J型指令

  • 无条件跳转
    • j address;
    • PC ← (PC+4) ∨ (address<<2)
  • 调用与联接
    • jal address;
    • $Ra ← PC + 4 （保存地址）
    • PC ← (PC+4) ∨ (address<<2)

CPU

CPU的组成与功能

冯诺依曼架构
计算机由以下部分组成：

  • 运算器
  • 控制器
  • 存储器
  • 输入设备
  • 输出设备

其中运算器和控制器构成了CPU：

  • 运算器：数据加工
    • 通用寄存器
    • 状态寄存器
    • ALU
  • 控制器：程序执行、指令执行
    • 取指令（MEM[PC++]→IR）
    • 执行指令：
      • 指令字→控制信号序列
      • 信号序列→数据通路

主要寄存器

  • PC：程序计数器
  • IR：指令寄存器
  • AR：地址寄存器
  • DR：数据寄存器
  • AC：累加寄存器
  • PSW：程序状态字

除PC外均为可选。

数据通路

数据通路是执行部件间传送信号的路径，不同指令或同一指令在不同阶段的数据通路不同。

分类：共享通路（总线）、专用通路。

基本结构

  • 寄存器A → 组合逻辑 → 寄存器B。
  • 专用通路：类似于多总线结构，并发高、设计复杂
  • 总线结构：各部件通过总线数据传输
    • 单总线结构：所有部件的输入和输出都通过一根总线传输，使用多个锁存器，时钟周期长
    • 多总线结构：不同部件的不同输入输出使用不同总线，减少锁存器数量，时钟周期短


对于单总线的连接方式来说，ALU只能有一端与总线相连，因为两端相连就必会发生冲突。所以另一段要设计一个暂存器，先把数据放入暂存器，暂存器再与总线相连。

典型数据通路：访存通路（内存）、总线传输（总线）、运算通路（ALU）。

控制器设计

控制器（control unit） 是根据指令、时钟信号和反馈信号输出相应控制信号序列的逻辑电路。控制信号是诸如RegWrite等使能控制部件的信号。

硬件布线控制器

时钟周期设计：

  • 定长指令周期：取最长指令的机器周期进行作为同步
  • 变长指令周期：根据每条指令的机器周期分别设计同步，较复杂

传统三级时序系统：

  • 三级时序：
    • 时钟脉冲
    • 状态周期电位（取指+执行）
    • 节拍电位
  • 时序产生器状态机：
    • 八个机器周期依次对应八个按序转移的状态，使用三位状态机
    • 状态寄存器+状态机组合逻辑（三位状态机变化）+时序产生器（状态机到具体信号）

硬布线控制器是纯组合逻辑的设计，基于时序系统。

Cn=∑m,i,k,j=(Im⋅Mi⋅Tk⋅Bj)C_n=\sum_{m,i,k,j}=(I_m\cdot M_i\cdot T_k\cdot B_j) Cn​=m,i,k,j∑​=(Im​⋅Mi​⋅Tk​⋅Bj​)

ImI_mIm​ 表示执行的指令，MiM_iMi​ 表示取指、执行周期之一，TkT_kTk​ 表示节拍电位。举例：

Read=MIF⋅(T2+T3)+LOAD⋅MEX⋅(T2+T3)Read=M_{IF}\cdot(T_2+T_3)+LOAD\cdot M_{EX}\cdot (T_2+T_3) Read=MIF​⋅(T2​+T3​)+LOAD⋅MEX​⋅(T2​+T3​)

表示Read控制信号在取指周期的第二、三个节拍电位和执行LOAD指令的第二、三个节拍电位给出。

固定周期硬布线控制器设计过程

  1. 设计三级时序产生器：所有指令固定机器周期数、节拍数
  2. 列出所有机器指令的指令周期流程图，明确每个节拍的控制信号
  3. 找出产生同一微操作控制信号的条件
  4. 写出各微操作控制信号的布尔表达式
  5. 化简各表达式
  6. 利用组合逻辑电路实现

现代时序系统：
为了节约固定周期中浪费的电位节拍，可以对状态进行更严格的区分，将所有指令执行的不同阶段都用一个状态来表示，扩展了仅用节拍电位和周期类型表示状态的传统三位状态机。

  • 时序产生器：时序状态转移不仅与现态时序有关，也与指令和反馈信号有关

可变周期硬布线控制器设计过程

  1. 列出所有机器指令的指令周期流程图，明确每个节拍的控制节奏
  2. 绘制指令执行状态转换图
  3. 根据状态转换图构建状态机真值表，实现有限状态机组合逻辑
  4. 实现控制器组合逻辑电路

微程序控制器

硬布线控制器使用组合逻辑，快但是繁琐且难以修改。微程序控制器解决了这些问题。

微指令：一组并发的控制信号，存储在控制存储器里。一条机器指令对应多条微指令。

微指令构造：[操作控制字段|判断字段|下址字段]

  • 操作控制字段：
    • 每一位对应一个控制信号（也称微命令），可同时给出多个操作信号
  • 顺序控制字段：
    • 包括判别逻辑和下址字段
    • 判别逻辑为0，下一条微指令地址从下址字段取出，否则按约定规则生成

微地址寄存器（uAR）：存放微指令地址的寄存器。微地址通过微程序入口地址和下址字段、判别字段组合得到。

微程序设计
  • 一条微指令对应一个时钟周期
  • 操作控制字段的信号在该时钟周期有效
  • 指令需要多少时钟周期就包含多少微指令

举例：取指指令

  1. (PC) → AR，(PC) →X
    • 控制信号：PCout、ARin、Xin
    • 微指令地址：0000
    • 下址字段：0000
    • 判别字段：00
    • 接下来两个微指令类似
  2. 最后一个微指令：
    • 下址字段：XXXX
    • 判别字段：10，表示下一条指令地址不由下址字段给出，根据事先约定的规则寻找下一条指令的入口
微指令格式

直接表示法：

  • 操作控制字段每一位对应一个控制信号
  • 简单直观，便于连接控制
  • 微指令长度太长
  • 解决缺点的方法：
    • 使用编码表示法
    • 去掉下址字段，uPC=uPC+1
    • 改水平型微指令为垂直型微指令

互斥型微命令：不能同时给出的微命令，例如输出到总线的系列out命令和计算型类微命令，没必要每一位对应一个微命令。
相容性微命令：可以同时给出的微命令。

编码表示法：

  • 将互斥型微命令压缩为小位数的编码，解码时通过译码器恢复原始微命令。
  • 需要预留一个状态，表示不给出这类指令的任何一个信号（例如七个信号对应一个三八译码器）

垂直型微指令：
将操作控制字段改写为执行指令的类别，大大减少了操作控制字段的长度。缺点是译码复杂，需要译码电路，基本被淘汰。

流水线

流水线（pipeline） 是一种实现多条指令重叠执行的技术。

通常，一个指令包括如下5个处理步骤：

  1. IF：从指令存储器中读取指令
  2. ID：指令译码的同时读取寄存器
  3. EX：执行操作或计算地址
  4. M：从数据存储器中读取操作数
  5. WB：将结果写回寄存器

使用流水线方式，每一个流水级（pipeline stage）都只花费一个时钟周期的时间，因此时钟周期必须满足最慢操作的需要。

理想状态下，执行执行时间流水线=指令执行时间非流水线/流水线级数

流水线冒险

流水线中有这样一种情况，在下一个时钟周期中下一条指令不能执行，这种情况称为冒险（hazard）。

  1. 结构冒险
    • 定义：硬件不支持多条指令在同一时钟周期下运行
    • 解决方法：
      • 缺少寄存器：增加寄存器
  2. 数据冒险
    • 定义：无法提供指令所需的数据，导致指令不能执行
    • 产生原因：一条指令依赖另一条还在流水线中的指令
    • 解决方法：
      • 旁路（bypassing） 从内部资源中直接提前得到缺少的运算项。
        • 在取数-使用型冒险 中，依然需要阻塞一个步骤。
  3. 控制冒险
    • 定义：决策依赖于一条指令的结果。例如，beq将设置PC，进而影响下一步取值操作。
    • 解决方法：
      • 阻塞
      • 预测
