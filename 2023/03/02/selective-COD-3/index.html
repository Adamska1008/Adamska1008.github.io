指令系统

指令概述及格式

基本概念：

  • 计算机语言中的基本单词称为指令，一台计算机的全部指令称为指令集（instruction set）。
  • 指令字长：指令中包含的二进制位数
    • 多字长指令：解决寻址较大存储空间的问题，多次取指影响速度
    • 等长指令：指令长度固定

指令分类：

  • 根据计算机层次结构：
    • 高级语言指令
    • 汇编语言指令
    • 机器语言指令
  • 根据指令中地址码字段个数分类
    • 三地址指令：[OP|A1|A2|A3] →(A1)OP(A2)=(A3)
    • 二地址指令：[OP|A1|A2] →(A1)OP(A2)=(A1)
    • 一地址指令：[OP|A1] →(AC)OP(A1)=(A1)
    • 零地址指令：[OP]，如停机、空操作、堆栈指令
  • 根据指令中操作数的物理位置分类：[OP|A1|A2]
    • 存储器-存储器（SS）型
    • 寄存器-寄存器（RR）型
    • 寄存器-存储器（RS）型
  • 根据指令的功能分类：
    • 传送指令：MOV、PUSH/POP、IN/OUT等
    • 定点算术运算指令：ADD、SUB、INC、CMP、MUL等
    • 位运算指令：NOT、AND、OR、SHL、SAL等
    • 控制转移指令：JMP、JNE、CALL、RET

指令格式：

  • 操作码字段：指令完成的功能
    • 定长操作码位数与最大指令数量有关：Lop=⌈ log2n ⌉
    • 变长操作码向不用的地址码字段扩展
  • 地址码字段：
    • 作用和影响与寻址方式和位数有关

寻址方式

指令寻址：
使用程序计数器（program counter） 保存当前运行的指令地址。每执行一条指令，PC加1。控制转移指令可能修改PC实现跳跃寻址。

数据寻址：
记地址E处的数据为S=(E)，操作数为D。

  • 立即数寻址：地址码字段是数据本身
    • 例：MOV AX, 200H
    • 特点：
      • 数据与指令一并读入寄存器，速度快
      • 便于程序设计
      • 数据大小受字段位控制
  • 寄存器寻址：操作数在寄存器内部
    • 例：MOV AX, BX
    • 特点：
      • 操作数在寄存器中，速度快
      • 能访问的数据大小与计算机字长有关
      • 地址字段位数与计算机通用寄存器数量相关：⌈ log2n ⌉
  • 直接寻址：地址码字段为操作数在内存中的地址
    • E=D, S=(D)
    • 例：MOV AX, [200H]
    • 特点：
      • 需要访问主存，速度慢
      • 地址字段位数决定了访问空间大小
  • 间接寻址：地址码是操作数主存地址的地址
    • E=D, S=((D))
    • 例：MOV AX, I[200H]
    • 特点：
      • 解决了直接寻址下地址字段位数限制访问范围大小的问题
      • 访问主存两次，太慢
  • 寄存器间接寻址：地址码是寄存器编号
    • E=®，S=(®)
    • 例：MOV AX, [BX]
    • 特点：
      • 解决了直接寻址下地址字段位数限制访问范围大小的问题
      • 访问主存一次
  • 相对寻址：以PC为基地址，地址码为偏移量，相加得到地址
    • E=D+(PC)
    • 例：JMP 200
    • 特点：
      • 节省地址位数
      • 由于PC的值在取指阶段完成修改，E需要额外增加
  • 基址寻址：指定一个基址寄存器B
    • E=D+(B), S=(E)
    • 例：MOV AX 32[B]
    • 特点：
      • 使用基址寻址可以访问更大空间
      • 对某一程序，基址设定后不变，通过修改D访问数据
      • 访问一次内存
  • 变址寻址：指定一个变址寄存器X，内容可以改变
    • E=D+(X)，S=(E)
    • 例：MOV AX, 32[SI]
    • 特点：
      • 不改变指令即可改变数据的有效地址，常在循环中使用

某计算机采用双字节长指令，内存基于字节寻址，指令数据用补码表示，PC的值在取指阶段完成修改

  1. 若采用相对寻址，当前地址为2003H，要求转移后目标地址为200AH，则该指令地址码为多少。
  2. 若采用相对寻址，当前地址为2003H，要求转移后目标地址为200AH，则该指令地址码为多少。
  1. 200AH-(2003H+2)=5H
  2. 2001H-(2008H+2)=-9=F7H

指令格式设计

主要内容：

  • 根据指令数量以及是否支持操作码扩展，确定操作码字段的位数
  • 根据对操作数的要求确定地址码字段的个数
  • 根据寻址方式，确定寻址方式字段
  • 确定采用定长还是变长指令

字长16位，主存64K，指令单字长单地址，80条指令，寻址方式有直接、间接、相对、变址

80条指令 → OP字段需要7位
4种寻址方式 → 寻址方式特征位2位
单地址位长度：16-7-2=7位
指令格式：[OP|X|A]

MIPS指令集

MIPS是精简指令集RISC（Reduced Instruction Set Computer），有以下特点：

  • 简单的Load/Store结构
  • 易于流水线CPU设计
  • 易于编译器开发
  • 寻址方式、指令操作简单

指令格式

  • R型指令：Rs、Rt为源操作数，Rd为目标操作数
  • I型指令：
    • Load/Store：Rs和立即数为源操作数，Rt为目标操作数
    • 条件转移：Rs、Rt均为源操作数
  • J型指令：26位立即数作为跳转目标地址的部分地址

MIPS寄存器

寄存器名 寄存器编号 用途说明
$s0 0 保存固定常数0
$at 1 汇编器临时变量
$v0~$v1 2~3 子函数调用返回结果
$a0~$a3 4~7 函数调用参数1~3
$t0~$t7 8~15 临时变量，函数调用时不需要保存和恢复
$s0~$s7 16~23 函数调用时需要保存和恢复的寄存器变量
$t8~$t9 24~25 临时变量，函数调用时不需要保存和恢复
$k0~$k1 26~27 中断、异常处理程序使用
$gp 28 全局指针变量（Global Pointer）
$sp 29 栈指针变量（Stack Pointer）
$fp 30 帧指针变量（Frame Pointer）
$ra 31 返回地址（Return Address）

MIPS寻址

MIPS指令集不单设置寻址方式字段。

  • R型指令：由op和func字段共同隐含说明当前的寻址方式

  • I型和J型指令：由op字段隐含说明当前的寻址方式

  • 立即数寻址：

    • I型指令：[OP|Rs|Rt|立即数]
    • 举例：addi s1, s2, 10 → $s1=$s2+10

  • 寄存器直接寻址：

    • R型指令：[OP|Rs|Rt|Rd|shamt|funct]
    • 举例：add $t0, $s1, $s2 → $t0=$s1+$s2

  • 基址寻址：

    • I型指令：[OP|Rs|Rt|立即数]
    • 立即数带符号扩展为32位
    • $Rs为基址寄存器，($Rs)+立即数=E, S=(E)
    • 使用基址寻址的指令：lw，sw，lh，sh，lb，lbu等

  • 相对寻址

    • I型指令（Rs、Rt用于条件判断）
    • 立即数带符号扩展成32位后左移2位
    • E=(PC)+立即数

  • 伪直接寻址（页面寻址）：

    • J型指令
    • 立即数左移2位
    • E=PC作为高四位+立即数

详细指令

R型指令
[OP|Rs|Rt|Rd|shamt|funct]

  • OP：操作码，所有R型指令OP全为零
  • Rs：寄存器编号，对应第一个操作数
  • Rt：寄存器编号，对应第二个操作数
  • Rd：寄存器编号，对应保存结果
  • shamt：常数，在移位指令中使用
  • funct：功能码，指定指令具体功能

三类指令：

  • 三寄存器R型指令：
    • add、sub、and、or、xor
    • 指令功能：$Rd ← $Rs op $Rt
  • 二寄存器R型指令
    • sll、srl、sra
    • $Rs=0、shamt=sa
    • 指令功能：$Rd ← &Rt shift sa
  • 一寄存器R型指令
    • jr
    • 只有$Rs非0
    • PC ← $Rs

I型指令
[OP|Rs|Rt|立即数]

  • OP：操作码
  • Rs：源寄存器
  • Rt：目的寄存器
  • Imm：立即数，第二个源操作数

四类指令：

  • 面向运算：
    • addi、andi、ori、xori
    • $Rt ← $Rs OP E(Imm)
    • addi使用符号扩展，其他指令使用0扩展
  • 面向访存
    • lw
      • lw Rt, Imm(Rs)
      • $Rt ← mem[$Rs+E(Imm)]
    • sw
      • sw Rt, Imm(Rs)
      • mem[$Rs+E(Imm)] ← $Rt
  • 面向数位设置
    • lui
      • lui Rt, Imm
      • $Rt ← Imm << 16 （空位补零）
  • 面向条件转移分支
    • beq（$Rs == $Rt）、bne（$Rs != $Rt）
    • if (cond) PC ← PC+E(Imm)<<2
    • 标准的PC相对寻址
    • Imm先符号扩展成32位，再左移两位

J型指令

  • 无条件跳转
    • j address;
    • PC ← (PC+4) ∨ (address<<2)
  • 调用与联接
    • jal address;
    • $Ra ← PC + 4 （保存地址）
    • PC ← (PC+4) ∨ (address<<2)
