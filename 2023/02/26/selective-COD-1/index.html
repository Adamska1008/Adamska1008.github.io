性能

性能的定义

响应时间：也叫执行时间，是计算机完成一项任务所需的总时间。

吞吐率：也叫贷款，一种度量性能的参数，表示单位时间内完成的任务量。

下面哪种改进能够增加吞吐率或减少响应时间？

  1. 将计算机中的处理器更换为更高速的型号
  2. 增加多个处理器分别处理独立任务。

减少响应时间一般都可增加吞吐率。方式1同时改进了响应时间和吞吐率，方式2不能使单个任务完成地更快，只能增加吞吐率。

性能常使用响应时间来度量。

性能x=1执行时间x性能_{x}=\cfrac{1}{执行时间_x} 性能x​=执行时间x​1​

性能的度量

时间常用来度量计算机的性能。对时间最直接的定义是墙钟时间。

CPU执行时间表示CPU为完成某任务花费的计算时间，不包括IO或等待其他程序运行的时间。它还可以进一步细分为用户CPU时间和系统CPU时间。用户感受到的是响应时间，不是CPU时间。

计算机用始终确定硬件中发生的事件，这种离散的时间间隔是时钟周期数。时钟周期长度用一个时钟周期的时间或其倒数时钟频率表示。

CPU的性能及其度量因素

度量CPU性能的底线是CPU执行时间。

CPU执行时间=CPU时钟周期数×时钟周期长度CPU执行时间=CPU时钟周期数\times 时钟周期长度 CPU执行时间=CPU时钟周期数×时钟周期长度

也可以写作

CPU执行时间=CPU时钟周期数时钟频率CPU执行时间=\cfrac{CPU时钟周期数}{时钟频率} CPU执行时间=时钟频率CPU时钟周期数​

某程序再时钟频率为2GHz的计算机A上运行需要10秒，在B计算机上运行需要6秒，但是时钟周期数是A上的1.2倍，求B的时钟频率。

指令的性能

计算机通过执行指令运行程序，执行时间依赖于指令数。

CPU时钟周期数=程序指令数×每条指令的平均时钟周期数CPU时钟周期数=程序指令数\times 每条指令的平均时钟周期数 CPU时钟周期数=程序指令数×每条指令的平均时钟周期数

CPI（Clock cycles Per Instruction）表示执行每条指令所需的平均时钟周期数，不同指令需要的时间可能不同。可以认为通过一个程序指令数不变，通过CPI可以比较相同指令集的不同实现方式。

假设有相同指令集的两种不同实现方式，计算机A的时钟周期为250ps，CPI为2.0；计算机B时钟周期为500ps，CPI为1.2，对同一个程序，哪个计算机运行地块？

经典CPU性能公式

CPU时间=指令数×CPI时钟频率CPU时间=\cfrac{指令数\times CPI}{时钟频率} CPU时间=时钟频率指令数×CPI​

或

CPU时间=指令数×CPI×时钟周期长度CPU时间=指令数\times CPI \times 时钟周期长度 CPU时间=指令数×CPI×时钟周期长度

MIPS

MIPS（Million Instructions Per Second，每秒百万条指令）是一种取代时间以度量性能的方法。对于一个给定的程序，MIPS表示为：

MIPS=指令数指令时间×106MIPS=\cfrac{指令数}{指令时间\times 10^6} MIPS=指令时间×106指令数​

MIPS是指令执行的速率，其所定义的性能与执行时间成反比。

计算机的算数运算

信息的表示

整数表示

假设一个整数数据类型（二进制）有w位，无符号数可以表示为[xw-1, xw-2,…,x1,x0]。采用最低有效位表示x0，最高有效位表示xw-1。它可以写成幂次之和的形式：

Uw(x⃗)=∑i=0w−1xi2iU_w(\vec{x})=\sum_{i=0}^{w-1}x_i2^i Uw​(x)=i=0∑w−1​xi​2i

用以表达无符号二进制数的值。

如果数值运算不能被有限位数的硬件表示，可能会产生溢出。一般来说，无符号数计算忽略溢出。

对于有符号数，需要考虑表示其符号。有三种方法表示有符号数：原码、反码、补码。

原码（Sign-Magnitude）：最高有效位是符号位，用来确定剩下的有效位是正权还是负权。

Sw(x⃗)=(−1)xw−1⋅(∑i=0w−2xi2i)S_w(\vec{x})=(-1)^{x_{w-1}}\cdot(\sum_{i=0}^{w-2}x_i2^i) Sw​(x)=(−1)xw−1​⋅(i=0∑w−2​xi​2i)

反码（One’s Complement）：按权值进行幂次相加计算，但是最高有效位的权值是−(2w−1−1)-(2^{w-1}-1)−(2w−1−1)。

Ow(x⃗)=−xw−1(2w−1−1)+∑i=0w−2xi2iO_w(\vec{x})=-x_{w-1}(2^{w-1}-1)+\sum_{i=0}^{w-2}x_i2^i Ow​(x)=−xw−1​(2w−1−1)+i=0∑w−2​xi​2i

这两种方式对于0会产生两种表达方式。实际上，所有现代机器基本使用补码。

补码（Two’s Complement）：最高位同样解释为负权值，但是值为−xw−1-x_{w-1}−xw−1​

Tw(x⃗)=−xw−12w−1+∑i=0w−2xi2iT_w(\vec{x})=-x_{w-1}2^{w-1}+\sum_{i=0}^{w-2}x_i2^i Tw​(x)=−xw−1​2w−1+i=0∑w−2​xi​2i

将一个整数转化为有符号数的方法：

  • 若整数为正，三种编码方式都增加一个最高位0，作为符号位，其他不变。
  • 若整数为负，三种编码方式都增加一个最高位1，作为符号位，其他不变
    • 原码其他位无变化
    • 反码其他位取反
    • 补码其他位取反，最低位加1

相较于其他两种编码方式，补码具有唯一表示零的方式，补码上的减法操作等于加上这个数的负值，使得加减法得到统一。

浮点数表示

计算机一般使用IEEE浮点标准表示浮点数。一个浮点数VVV被表示为V=(−1)s×M×2EV=(-1)^s\times M\times 2^EV=(−1)s×M×2E，其二进制编码中分块存储了s, M与E的数值。

  • 符号（sign） 决定了数值的正负
  • 尾数（significand） M是一个二进制小数，它的范围是1~2-ε或0~1-ε
  • 阶码（exponent） 的作用是对浮点数加权，这个权重是2的E次幂（可能是复数）。

浮点数的位划分为三个段。

  • 一个单独的符号位s直接表示符号s。
  • k位的阶码字段exp=ek-1…e0编码阶码E
  • n位小数字段frac=fn-1…f0编码位数M。

一个二进制编码被划分为：[s][exp][frac]。

对于32位编码，len(exp)=8、len(frac)=23；对于64位编码，len(exp)=11、len(frac)=52。

以32位浮点数为例，根据exp的值，被编码的值可以分成三种不同的情况：

  • 规格化的：exp ≠ 0 & exp ≠ 255
  • 非规格化的：exp = 0
  • 无穷大 & NaN：exp = 255
    • 无穷大：frac = 0
    • NaN：frac ≠ 0

规格化值：

  • 阶码：以*偏置（biased）*形式表示的有符号整数，值为E=e−BiasE=e-BiasE=e−Bias，其中e是原值无符号数，而Bias是一个等于2k-1的偏置值。
  • 小数字段：描述小数值f,0≤f<1f, 0\le f < 1f,0≤f<1。二进制表示的含义为0.fn-1…f0。尾数M定义为M=1+fM=1+fM=1+f。

非规格化值：

  • 阶码值：E=1−BiasE=1-BiasE=1−Bias。
  • 小数字段：M=fM=fM=f，这表示小数字段的值不包括隐含的1。
  • 用途：
    • 表示数值0：在规格化值中，总有M≥1M\ge 1M≥1，无法表达0。具体来说，浮点数0表示为所有二进制位都是0。
    • 表示非常接近0的数

特殊值：

  • 无穷：s=0时是+∞，s=1时是-∞。表示溢出的结果，如两个非常大的数相乘或除以零。
  • NaN（Not a number）：不是实数或无穷，例如−1\sqrt{-1}−1​。

再移动小数点，使之变成1.f的形式：1.010010011×24

数据校验

基本原理：增加冗余码（校验位）。

码距：同一编码中，任意两个合法编码之间不同二进制位数的最小值。

校验码通过增加冗余位，增大了码距。

码距可以衡量编码的检错和纠错能力：

  • 码距 ≥ e + 1：可检测e个错误
  • 码距 ≥ 2t + 1：可纠正t个错误
  • 码距 ≥ e + t + 1：可纠正t个错误，同时检测e个错误

奇偶校验

增加1位冗余码，使校验码（数据+冗余码）中1的个数满足奇/偶校验的要求。码距为2。

0001 →00011 （偶校验）
0001 →00010 （奇校验）

检错方法：按位异或。奇校验码期望值为0，偶校验码期望值为1。

特点：

  • 编码简单，效率高
  • 不能检测偶数位错误（不可靠，错误检测码）
  • 没有纠错能力

改进：双向奇偶校验。在传统奇偶校验的基础上，进行纵向的奇偶校验。

特点：

  • 可纠正一位错误
  • 可检测出某行某列的奇数位错误
  • 可检出一部分偶数位错误

CRC（循环冗余码）校验

设置r位校验位。假设有效信息为k位，需要满足N=k+r≤2r-1，其中2r-1表示可以表述的错误的数量，含义是可以表示的错误数量要大于总位数。

生成多项式G(x)：(r+1)位序列，发送方利用G(x)对被校验信息进行模2除运算，生成校验码。接收方利用G(x)对收到的编码做模2除运算，检测差错以及错误定位。

G(x)应满足的条件：

  • 最高位和最低位为1
  • 当CRC码（被传送信息）任何一位出错后，被G(x)除后余数不为0
  • 不同位发生错误，模2除运算后余数不同
  • 对不为0余数继续进行模2除运算，应使余数循环

模2运算：

  • 加/减运算：等价于异或
  • 除运算：基于模2减进行运算

CRC编码方法：

  • 根据待校验信息选择r的位数
  • 选择位数为r+1的生成多项式G(x)
  • 将待校验信息左移r位
  • 按模2除法除G(x)，得到r位余数即为冗余码
  • 用余数替换最后r位
  • 接收方使用G(x)对收到数据做模2运算，若余数为0则没有错误

CRC的纠错与检错：
不同位数出错时，对应的余数不同，但没有具体规律。采用不同多项式，相同位出错时，余数可能不同。

CRC具有循环特性。假如第k位出错，对计算后的余数末尾补零继续模2运算，得到的余数对应第k-1位出错时的余数。

利用循环特性，只需记住某一位（如第一位）出错时的余数，在循环运算时将校验码左移，当计算结果等于记住的余数时，修正第一位，即可正确修正错误位数。接下来只要继续循环使得纠正后的出错位回到原位。

海明校验

海明码校验同样需要满足N=k+r≤2r-1不等式，其含义与CRC相同。

记k+r位海明码含1, 2, 3, … k+r位，r位校验位记为Pi，分别位于k+r位海明码的第2i-1位上。

对于第k位的数据，必定可以将k二进制拆分为2的幂次和。这些幂即为校验该数据位的校验位的值。

将校验位对应数据位进行偶校验，即可得出校验位的值。

对于每一个校验位，设置指错字，指错字为该校验位与其对应数据位偶校验的值。所有指错字的组合即表示出错的具体位数。

特点：

  • 指错字为0，不一定无错
  • 一位错和两位错不能由指错字指出

溢出：
当运算结果超过硬件位数限制时，发生溢出。当两个符号相异的操作数相加时，结果小于任意一个操作数，故不会溢出；溢出只会发生在相同符号的操作数下。

对于补码，可以通过结果的正负来判断是否溢出

对于无符号数，加法溢出可以用ALU的进位表示，减法溢出则为进位信号取反。一般来说，可以忽略无符号数溢出。

定点数加减运算器

我们使用一位全加器（Full Adder）设计多位串行加法器（Binary Adder）。

将四位FA按从低到高的顺序，依次将低位的进位输出位连接到相邻高位的进位输入位，就构成了四位的加法器。

由定点数运算公式可得，两数差的补码等于被减数的补码加上减数的负数的补码。基于此设计加减运算器。

我们将减数每位的输入与一个操作位K异或。当操作位为1时，按位异或1可得到减数的反码。同时，这个操作位还连接了最低位全加器的进位，由于补码等于反码加1，故操作位为1时就得到了减数的反码，加法器就变成了减法器。

原码一位乘法

考虑竖式法求解乘法，可以将二进制乘法拆分为多个被乘数移位之后的和。为了优化求解，我们不使用多个ALU，只用一个ALU进行累加计算。

我们用一个寄存器存放乘数和结果，通过低位比特判断是否累加被乘数到高位。每次加完之后，将该寄存器右移1比特，将原本放置在高位的结果逐渐回归到低位。

定点数除法

类似于乘法，除法以减法为基础实现，基本思路是将被除数不断左移，减去除数。

在某一次减法中，需要判断左移的被除数（余数）是否大于除数，这需要通过一次减法判断，即试商：

  • 假如余数为正数，则商上1，将余数左移，继续比较
  • 假如余数为负数，商上0，并加除数恢复原来的值，继续左移比较
