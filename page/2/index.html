加成序列

满足如下条件的序列 XXX（序列中元素被标号为 1、2、3…m1、2、3…m1、2、3…m）被称为“加成序列”：

  1. X[1]=1X[1]=1X[1]=1
  2. X[m]=nX[m]=nX[m]=n
  3. X[1]<X[2]<…<X[m−1]<X[m]X[1]<X[2]<…<X[m−1]<X[m]X[1]<X[2]<…<X[m−1]<X[m]
  4. 对于每个 k（2≤k≤m）k（2≤k≤m）k（2≤k≤m）都存在两个整数 iii 和 jjj （1≤i,j≤k−11≤i,j≤k−11≤i,j≤k−1，iii 和 jjj 可相等），使得 X[k]=X[i]+X[j]X[k]=X[i]+X[j]X[k]=X[i]+X[j]。
    你的任务是：给定一个整数 nnn，找出符合上述条件的长度 mmm 最小的“加成序列”。

如果有多个满足要求的答案，只需要找出任意一个可行解。

输入格式

输入包含多组测试用例。

每组测试用例占据一行，包含一个整数 n。

当输入为单行的 0 时，表示输入结束。

对于每个测试用例，输出一个满足需求的整数序列，数字之间用空格隔开。

每个输出占一行。

数据范围

1≤n≤1001≤n≤1001≤n≤100

输入样例

输出样例

题解

迭代加深算法经典题。

首先分析一下如何搜索。题目要求每一个数都必须是前两个数的和，那么我们可以枚举每两个数来生成后一个数。

然后分析一下如何剪枝，首先是如何枚举两个数的和。为了保证每一层的分支尽量小，我们需要先枚举较大的和。其次，枚举可能有重复，比如1 2 3 4, 2+3=1+4=52 + 3 = 1 + 4 = 52+3=1+4=5，同一个数枚举了两次，所以需要开一个bool数组记录哪些数已经使用过了。

最后，我们分析一下这个题的特点。n的上界是100，很大，而且我们很容易构造出来一个数组1 2 3 4 ... 98 99 100，可见搜索过程中是有可能搜到100层的。但是，不难看出，题目中所需的最小数组长度其实是很小的，77才9层，所以答案层数明显小于上界层数，这个时候我们可以使用迭代加深，枚举深度上限来进行dfs，就能防止搜到过于巨大的状态导致TLE。

最大子序和

输入一个长度为 nnn 的整数序列，从中找出一段长度不超过 mmm 的连续子序列，使得子序列中所有数的和最大。

注意： 子序列的长度至少是 111。

输入格式

第一行输入两个整数 n,mn,mn,m。

第二行输入 nnn 个数，代表长度为 nnn 的整数序列。

同一行数之间用空格隔开。

输出格式

输出一个整数，代表该序列的最大子序和。

数据范围

1≤n,m≤3000001≤n,m≤3000001≤n,m≤300000

输入样例：

输出样例：

题解

O(n2)O(n^2)O(n2) 的最大子序和有两种写法，一个是dp，一个是前缀和，一般认为dp比前缀和好，因为前缀和可能爆int甚至long long，但是前缀和可以使用单调队列优化为 O(n)O(n)O(n)。

以样例为例，前缀和为1 -2 3 4 2 5。对于前缀和，我们实际上只关心一段序列中最大值与最小值之差的最大值（必须保证最小值在最大值之前）。我们看到最小值为 -2, 最大值为 5 ,且 -2 在 5 之前，那么最大子序列一定是 7 （忽略长度限制）。

这显然就是一个单调队列，队首为一段区间内的最小值的序号，队尾为最大值的序号。

玉米田

农夫约翰的土地由 M×NM×NM×N 个小方格组成，现在他要在土地里种植玉米。

非常遗憾，部分土地是不育的，无法种植。

而且，相邻的土地不能同时种植玉米，也就是说种植玉米的所有方格之间都不会有公共边缘。

现在给定土地的大小，请你求出共有多少种种植方法。

土地上什么都不种也算一种方法。

输入格式

第 111 行包含两个整数 MMM 和 NNN。

第 2..M+12..M+12..M+1 行：每行包含 NNN 个整数 000 或 111，用来描述整个土地的状况，111 表示该块土地肥沃，000 表示该块土地不育。

输出格式

输出总种植方法对 10810^8108 取模后的值。

数据范围

1≤M,N≤121≤M,N≤121≤M,N≤12

输入样例

输出样例：

题解

这个题很明显是一个状态压缩，我们定义 f[i][s] 为处理完前 i 行，且最后一行状态为 s 的情况的方法数。状态 s 种，1 为种，0 为没有种，比如状态101表示在第一、三块上种了，在第二块上没有种。

我们同样使用状态压缩记录土壤的贫瘠状况，不过对于01的定义方式与题目中相反，这样后面计算的时候会比较方便点。

首先，对于111之类的种法，由于已经有相邻的玉米了，所以无论如何都不可能转移到这种状态，所以我们可以预处理一下，排除所有相邻种植的状态。当 m=12m=12m=12 时，如果纯暴力枚举，需要枚举4096种状态，但事先排除一些状态后，我们可以发现合法状态只有377种，这便能优化时间。

如何进行状态转移呢？我们举一些例子。假如前一行状态为 0001，而本行的土壤状况为1000，那我们实际可种植的只有第二第三块。实际上就是把前两者按位或一下，得到1001，如果方案 s 在1的位置上都是0，那么便可以进行一个转移。如何判断呢？我们只需要按位与一下，判断是否为0。比如状态0100，按位与之后是0000，合法，状态1000，按位与之后是1000，不合法。

对于第 i 行的所有可能状态，枚举所有上一行的状态，能转移就更新，不能转移就跳过，时间复杂度是 O(n2n+1)O(n 2 ^{n+1})O(n2n+1)。做了第一段所述的优化之后，还能更快一点，实际复杂度是 12×377×377=1,705,54812 \times 377 \times 377=1,705,54812×377×377=1,705,548 这样子，是可以过的。

题解-tarjan-受欢迎的牛

受欢迎的牛

每一头牛的愿望就是变成一头最受欢迎的牛。

现在有 NNN 头牛，编号从 111 到 NNN，给你 MMM 对整数 (A,B)(A,B)(A,B)，表示牛 AAA 认为牛 BBB 受欢迎。

这种关系是具有传递性的，如果 AAA 认为 BBB 受欢迎，BBB 认为 CCC 受欢迎，那么牛 AAA 也认为牛 CCC 受欢迎。

你的任务是求出有多少头牛被除自己之外的所有牛认为是受欢迎的。

输入格式

第一行两个数 NNN,MMM；

接下来 MMM 行，每行两个数 AAA,BBB，意思是 AAA 认为 BBB 是受欢迎的（给出的信息有可能重复，即有可能出现多个 AAA,BBB）。

输出格式

输出被除自己之外的所有牛认为是受欢迎的牛的数量。

数据范围

1≤N≤104,1≤N≤10^4,1≤N≤104,
1≤M≤5×1041≤M≤5×10^41≤M≤5×104

输入样例

输出样例

样例解释

只有第三头牛被除自己之外的所有牛认为是受欢迎的。

题解

首先，如果使用朴素解法，对于一个点，搜索全图，看看是否被所有其他牛喜爱，这种算法是 O(n2)O(n^2)O(n2) 的，会超时。

考虑在一个拓补图中搜索。假如一个图是拓补图，显然，最多只有一头牛会被所有牛喜爱，这头牛必定是拓补序的最末端，即出度为 000 的点，同时必须满足只有这一个点出度为 000。如果有两个点满足该要求，显然这两个点不能互相喜爱。

虽然题目中没有保证图是拓补的，但是，显然，图中的所有强连通分量是拓补的。这题可以转化为求的拓补序末的强连通分量中点的数量，且要求出度为 000 的强连通分量只能有一个。
