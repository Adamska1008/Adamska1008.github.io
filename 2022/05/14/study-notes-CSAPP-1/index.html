CSAPP笔记（一）—— 信息的表示和处理

发表于 2022-05-14 更新于 2022-05-16 分类于 CSAPP学习笔记

信息存储

大多数计算机使用 字节（byte） 作为最小可寻址的内存单位。机器级程序将内存视为一个非常大的字节数组，称为 虚拟内存（virtual memory） 。内存的每个字节都由一个唯一的数字来标识，称为它的地址（address）。所有可能地址的集合就称为 虚拟地址空间（virtual address space）。

十六进制表示法

由于二进制表达法非常冗长，故常用 十六进制（hexadecimal）数，来表示位模式。显然，二进制到十六进制的转化非常容易。只需要以四比特为单位转化即可。在大部分编程语言中，使用0x标识十六进制数，如0xFA1D37B。

字数据大小

每台计算机都有一个 字长（word size） 来标识指针数据的标称大小。对于一个字长为 www 位的机器而言，其虚拟地址的范围是 0∼2w−10 \sim 2^w-10∼2w−1，程序最多访问 2w2^w2w 个字节。这是由于，www 位二进制数最多可以表示 2w2^w2w 个数字。

目前常用的字长为32位或64位。

寻址和字节顺序

对于跨越多字节的程序对象，我们必须建立两个规则：

  1. 这个对象的地址是什么
  2. 在内存中如何排列这些字节

一般而言，多字节对象的字节序列是连续的，对象的地址为所有字节中的最小地址。

排列表示一个对象的字节由两个通用的规则，大端法（big endian） 与 小端法（little endian）。

考虑一个 www 位的整数，其位表示为 [xw−1,xw−2,⋅⋅⋅,x1,x0][x_{w-1}, x_{w-2}, ···,x_1, x_0][xw−1​,xw−2​,⋅⋅⋅,x1​,x0​]，其中 xw−1x_{w-1}xw−1​ 是最高有效位， x0x_{0}x0​ 是最低有效位。

假设 www 是 8 的倍数，，这些位就能被分组成为字节，其中最高有效字节包含位 [xw−1,xw−2,⋅⋅⋅,xw−8][x_{w-1}, x_{w-2}, ···,x_{w-8}][xw−1​,xw−2​,⋅⋅⋅,xw−8​]，而最低有效字节包含位 [x7,x6,⋅⋅⋅,x0][x_7, x_6, ···,x_0][x7​,x6​,⋅⋅⋅,x0​]。

最低有效字节在最前面的方法，称为小端法，最高有效字节在最前面的方法， 称为大端法。

例如，0x01234567使用小端法表示为：

0x100 0x101 0x102 0x103
… 01 23 45 67 …

而大端法则为：

0x100 0x101 0x102 0x103
… 67 45 23 01 …

大多数情况下，字节顺序对程序员不可见。但三个情况下，需要注意字节顺序的问题

  1. 网络传输数据时，要注意大端小端码的转换。因为接收时字节会反序。
  2. 阅读字节序列时，大端码需要反序阅读，如机器码中即时值0x200b43在字节序列中可能为43 0b 20 00。
  3. 使用强制类型转换来规避正常的类型系统时。

扩展一个数字的位表示

要将一个无符号数转化为一个更大的数据类型，只需要在开头添加0，这种运算被称为 零扩展（zero extension） 。

要将补码数字转换为一个更大的数据类型，可以执行 符号扩展（sign extension） ，在表示中添加最高有效位的值。

IEEE浮点表示

前文中的定点表示法不能有效地表示非常大的数字。而IEEE浮点标准用 V=(−1)s×M×2EV=(-1)^s \times M \times 2^EV=(−1)s×M×2E 的形式来表示一个数。

  • 符号（sign） sss 决定这数是负数（s=1s=1s=1）还是正数（s=0s=0s=0），而对于数值0的符号位解释作为特殊情况处理。
  • 尾数（significand） MMM 是一个二进制小数，它的范围是 1∼2−ε1 \sim 2-\varepsilon1∼2−ε 或是 0∼1−ε0 \sim 1-\varepsilon0∼1−ε。
  • 阶码（exponent） EEE 的作用是对浮点数加权，这个权重是 2 的 EEE 次幂（可能是负数）。

将浮点数的位表示划分为三个字段，分别对这些值进行编码：

  • 一个单独的符号位 sss 直接编码符号 sss 。
  • kkk 位的阶码字段 exp=ek−1⋅⋅⋅e1e0exp=e_{k-1}···e_1e_0exp=ek−1​⋅⋅⋅e1​e0​ 编码阶码 EEE。
  • nnn 位小数字段 frac=fn−1⋅⋅⋅f1f0frac=f_{n-1}···f_1f_0frac=fn−1​⋅⋅⋅f1​f0​ 编码尾数 MMM，但是编码出来的值也依赖于阶码字段的值是否等于 0 。

在单精度浮点格式（C语言中的 float）中，sss、expexpexp 和 fracfracfrac 字段分别为 111 位、k=8k=8k=8 位，和 n=23n=23n=23 位，得到一个32位的表示。

在双精度浮点格式（C语言中的 double 中），sss、expexpexp 和 fracfracfrac 字段分别为 111 位、k=11k=11k=11 位，和 n=52n=52n=52 位，得到一个64位的表示。

给定位表示，被编码的值可以分为三种不同的情况。

规格化的值

当 expexpexp 的位模式既不全为 0（数值 0 ），也不全为 1（单精度数值为255，双精度数值为2047）时，阶码的值为 E=e−BiasE=e-BiasE=e−Bias，其中 eee 即为对应的二进制无符号数 ek−1⋅⋅⋅e1e0e_{k-1}···e_1e_0ek−1​⋅⋅⋅e1​e0​，而 BiasBiasBias 为 2k−1−12^{k-1}-12k−1−1，单精度为127，双精度为1023。指数的取值范围，对单精度是 −126∼+127-126 \sim +127−126∼+127 ，对单精度是 −1022∼+1023-1022 \sim +1023−1022∼+1023 。

小数字段 fracfracfrac 被解释为描述小数值 fff ，其中 0≤f<10 \leq f < 10≤f<1 ，其二进制表示为 0.fn−1⋅⋅⋅f1f00.f_{n-1}···f_1f_00.fn−1​⋅⋅⋅f1​f0​。

尾数定义为 M=1+fM=1+fM=1+f。

非规格化的值

当阶码域全为 0 时，所表示的数是非规格化形式。在这种情况下，阶码值是 E=1−BiasE=1-BiasE=1−Bias，而尾数的值是 M=fM=fM=f，不包含隐含的开头的1。

非规格化数有两个用途：

  1. 提供了一种表示数值 000 的方法。当所有二进制位皆为 000 时，即表示浮点数 000。特别的是，符号位为 1 时，认为是 −0-0−0，在少数情况下会区别对待。
  2. 表示那些非常接近于 0.00.00.0 的数字。称为 逐渐溢出（gradual overflow），其中，可能的数值分布均匀地接近于 0.00.00.0。

特殊值

当阶码域全为 111 时出现的数为特殊值。当小数域全为 000 时，得到的值表示无穷，当 s=0s=0s=0 时是 +∞+\infty+∞，当 s=1s=1s=1 时是 −∞-\infty−∞ 。当我们把两个非常大的数相乘，或除以零时，无穷能够表示溢出 的结果。当小数域为非零时，结果值被称为 NaN，即“不是一个数（Not a number）”的缩写。
