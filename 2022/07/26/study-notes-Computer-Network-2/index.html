计算机网络笔记（二）-应用层

发表于 2022-07-26 更新于 2022-08-10 分类于 《计算机网络：自顶向下方法》学习笔记

计算机网络-应用层知识点

网络应用的体系结构：

  1. 客户机/服务器结构（Client-Server，C/S）
  2. 点对点结构（Peer-to-peer，P2P）
  3. 混合结构（Hybrid）

客户机/服务器结构

服务器特点

  1. 7*24小时提供服务
  2. 永久性访问地址/域名
  3. 利用大量服务实现可扩展性

客户机特点

  1. 与服务器通信，使用提供的服务
  2. 间歇性接入网络
  3. 可能使用动态ip地址
  4. 不会与其他客户机直接通信

P2P结构

  1. 没有永远在线的服务器
  2. 任意端系统/节点之间可以直接通讯
  3. 节点间歇性接入网络
  4. 可能改变ip地址

网络应用进程通信

同一主机上的进程通过操作系统提供通信机制，不同主机上运行的进程使用消息交换，客户机进程是发起通信的进程，服务器进程是等待通信请求的进程。

Socket
进程通过Socket抽象使用下层协议提供的服务。

进程寻址
使用ip + 端口号，ip唯一标识了主机，端口号唯一标识进程。

应用层协议

公开协议
由RFC定义，例如http，遵循相同协议的不同网络应用可以互操作。=

私有协议
多数P2P文件使用私有协议

内容

  • 类型：请求消息、响应消息
  • 语法：消息中有哪些字段，如何描述
  • 语义：信息的含义
  • 规则：何时/如何发送并相应消息

Web 与 HTTP

特征：网页，网页互相链接

网页

  • 对象：HTML文件、JPEG图片、视频文件、动态脚本等
  • 基本HTML文件：包含对其他对象引用的链接

对象的寻址：

  • URL(Uniform Resource Locator)：统一资源定位器
  • 基本格式：Scheme://host:port/path（协议，主机号，端口号，路径）

协议

超文本传输协议：HyperText Transfer Protocol

C/S结构：

  • 客户-Browser：请求、接收、展示Web对象
  • 服务器-Web Server：响应客户的请求，发送对象

传输层协议：TCP

  • 服务器在80端口等待客户请求
  • 浏览器发起到服务器的TCP连接（创建套接字Socket）
  • 服务器接受来自浏览器的TCP连接
  • 浏览器与服务器交换HTTP消息
  • 关闭TCP连接

特点：无状态，服务器不维护任何有关客户端过去所发请求的信息。

连接类型

  1. 非持久性连接
    • 每个TCP连接最多允许传输一个对象
    • HTTP 1.0 使用
  2. 持久性连接
    • 每个TCP连接允许传输多个对象，发送响应后，服务器保持TCP连接的打开
    • HTTP 1.1 使用

连接过程：
非持久性连接：

  1. 发起、建立TCP连接：1个RTT
  2. 发送HTTP请求消息到响应消息的前几个字节到达：1个RTT
  3. 响应消息中所含的文件/对象传输时间

无流水的持久性连接：

  1. 客户端只有收到前一个响应才发送新的请求
  2. 每个对象耗时1个RTT

带有流水机制的持久性连接

  1. 客户端只要遇到一个引用对象就尽快发出请求
  2. 同时请求的所有对象只耗费1个RTT

消息格式

两类消息：请求消息（request）、响应消息（response）。使用ASCII码，人直接可读。

请求消息：
第一行：request line: 请求命令+url+http版本

前几行：header lines: 键值对，包括Host,User-agent(浏览器类型),Connection,Accept-language

空行，表示头部结束。
最后是请求的数据（Entity Body）。

请求的数据可以使用POST或GET上传。

  • POST方法：例如表格中，将消息体放在Entity Body中上传
  • GET方法：输入信息通过URL字段上传

方法的类型：GET、POST、HEAD、PUT、DELETE。其中HEAD请求只返回消息，不返回请求对象。PUT将消息体中的文件上传到URL字段指定的路径。DELETE删除URL字段指定的文件。

响应消息：
第一行：status line：协议+状态码+状态语句
前几行：header lines: 键值对，包括Connection，Date，Server，Last-Modified，Content-Length，Content-Type。
空行区分
最后是请求的文件。

Cookie

某些网站为了辨别用户身份，进行session跟踪而存储在用户本地终端上的数据（通常经过加密）。

Cookie的组件：

  • HTTP响应消息的cookie头部行
  • HTTP请求消息的cookie头部行
  • 保存在客户端主机上的cookie文件，由浏览器管理
  • Web服务器端的后台数据

Web缓存/代理服务器技术

功能：在不访问服务器的前提下，满足客户端的HTTP请求

  • 缩短客户请求的响应时间
  • 减少机构/组织的流量
  • 在大范围内（Internet）实现有效的内容分发

用户设定浏览器通过缓存进行Web访问。浏览器向缓存/代理服务器发送所有的HTTP请求

  • 如果所请求对象在缓存中，缓存返回对象
  • 否则，缓存服务器向原始服务器发送HTTP请求，获取对象，然后返回给客户端并保存该对象

缓存既充当客户端，也充当服务器，一般由ISP架设。

条件性GET方法
在HTTP请求消息中声明所持有版本的日期：If-modified-since:<data>，如果服务器中缓存的版本是最新的，则响应消息中不包含对象，返回条件码304: Not Modified。

Email应用

构成：

  • 邮件客户端：
    • 读写Email消息
    • 与服务器交互，收、发Email消息
  • 邮件服务器
    • 邮箱：存储发给该用户的Email
    • 消息队列：存储等待发送的Email
  • SMTP协议（Simple Mail Transfer Protocol）
    • 邮件服务器之间传递消息所使用的协议
    • 客户端：发送消息的服务器
    • 服务端：接收消息的服务器

SMTP

  • 使用TCP
  • 端口25
  • 传输过程的三个阶段
    • 握手
    • 消息的传输
    • 关闭
  • 命令/响应交互模式
    • 命令：ASCII文本
    • 响应：状态代码和语句
  • Email消息只能包含7位ASCII码
  • 使用持久性连接
  • 要求消息必须由7位ASCII码构成
  • SMTP服务器利用CRLF确定消息的结束。
