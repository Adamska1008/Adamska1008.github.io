计算机网络笔记（六）- 链路层

发表于 2022-12-28 更新于 2023-01-01 分类于 《计算机网络：自顶向下方法》学习笔记

计算机网络-链路层知识点。

概述

链路层和网络层的关系：

  • 网络层：将传输层报文从源主机传递到目的主机
    • 能够在各段链路层提供异构服务的情况下，完成端到端的工作
  • 链路层：将网络层数据报从一个节点 传送到下一个节点（单段链路）
    • 不同的链路采用不同的链路层协议，提供的服务不同。

节点：主机和路由器
链路：连接沿通信路径的相邻节点的路径是链路，分为有线和无线
帧：第二层分组的命名，封装数据报

链路层信道类型

分为广播链路和点对点通信链路两种。

广播链路：许多主机被连接到相同的通信信道，需要媒体访问协议（MAC） 来协调传输和避免“碰撞”（冲突）。

  • 传统的以太网
  • 802.11无线局域网
  • 卫星网

点对点通信链路：直线链接两个节点的链路，每一端有一个节点。访问控制简单，主要解决问题：成帧，可靠数据传输、差错检测和流量控制。

链路层服务

  • 成帧，链路访问;
    • 将数据报封装进帧，加上首部和尾部。
    • 如果共享媒体，媒体访问控制（MAC）协议协调多路访问
    • 位于帧首部的MAC地址表示源、目的地（不同于IP）
  • 相连节点间的可靠交付
    • 在比特差错低的链路很少使用（光纤，双绞线）
    • 无线链路（高差错率）：差错检测，纠错
  • 流量控制：
    • 相邻发送和接收节点间的步调一致
  • 差错检测：
    • 差错由信号衰减、噪声所致
    • 接收方检测差错的存在
      • 信号发送方负责重传
  • 纠错：
    • 接收方识别和纠正比特差错，而不采取重传
  • 半/全双工

差错检测与纠错

发送方发送数据帧+冗余数据（EDC，差错检测和纠正比特），接收方检查信息数据和冗余数据的关系，发现差错。

奇偶校验

设置一个奇偶校验位，使得数据中1的个数为奇数或偶数。

  • 单比特奇偶校验：在数据末尾添加0或1，错误率50%
  • 二维比特奇偶校验：数据D被划分为i行j列，对每行每列计算奇偶值，产生i+j+1奇偶比特

校验和（检查和）

发送方：

  • 将段内容作为16比特整数序列来处理
  • 校验和：段内容相加（求和，回卷，求反）
  • 将校验和的值放入UDP校验和字段

接收方：

  • 计算接收到段的检查和
  • 检查是否各段之和全为1：
    • NO-检测到差错
    • YES-没有检测到差错（依然可能有错）

循环冗余码校验

收发双方约定一个r+1比特的生成多项式G(x)，如100000111表示x8+x2+x1+1（CRC-8标准）。

发送方用位串D和G(x)运算得到r比特的校验和R，在帧的末尾加上校验和，使带校验和的DR能被G(x)模2运算整除。

接收方之后用G(x)除以多项式，若有余数则传输出错。

模2运算

加法不进位，减法不借位，加减法即是按位异或。乘除法中涉及加减的部分不进位。

计算R

我们需要计算得到R，使得DR能被G(x)整除，即：

D×2rXOR R=nGD×2r=nG XOR RD\times 2^r XOR\ R = nG \\ D\times 2^r = nG\ XOR\ R D×2rXOR R=nGD×2r=nG XOR R

考虑到有 G>RG>RG>R，等价于

D×2r=nG+RD\times 2^r = nG+R D×2r=nG+R

即

D×2rmod  G=RD\times 2^r\mod G=R D×2rmodG=R

所以计算R的方式就是求(D << r) % G。

多路访问协议（MAC）

多路访问协议是决定节点怎样共享信道的分布式算法。

理想的多路访问协议有如下特点：
对于一个速率R的信道

  • 效率：当一个节点要传输，它能够以速率R发送
  • 公平：当M节点要传输，每个能以平均速率R/M发送
  • 全分散：
    • 无特殊节点来协调传输
    • 无同步时钟、时隙
  • 简单

MAC协议可以分为三大类：

  • 信道划分：
    • 将信道划分为较小的“段”（时隙、频率、编码），为节点分配一部分专用
  • 随机访问：
    • 不划分信道，允许碰撞
    • 从“碰撞”恢复
  • 轮流协议：
    • 节点轮流，无碰撞
    • 兼顾效率和公平
    • 单点失效，轮流时延

信号划分MAC协议

TDMA 时分多路访问

  • “周期性”访问信道
  • 每个站点在每个周期中获得固定长度时隙（长度=分组传输时间）
  • 不使用的时隙空闲
  • 特点：
    • 避免冲突，公平
    • 节点速率有限，效率不高

FDMA 频分多路访问

  • 信号频谱划分为频带
  • 每个站点分配固定的频带
  • 频带中未使用的频带空闲
  • 特点：与TDM类似

随机访问协议

  • 以信道全部速率R发送
  • 没有事先的节点间协调
  • 两个或更多传输节点可能碰撞，协议定义了：
    • 如何检测碰撞
    • 如何从碰撞中恢复

ALOHA

ALOHA采用星型拓补结构，中心主机通过下行通道向二级主机广播分组，二级主机通过上行信道向中心主机发送分组。

时隙ALOHA

假定：

  • 所有帧具有相同长度
  • 时间划分为等长时隙，能够传输一个帧
  • 节点仅在时隙开始时传输帧
  • 节点是同步的

操作：

  • 当节点获得新帧，将在下一个时隙中传输
  • 无碰撞时，节点能够发送新帧
  • 如果碰撞，节点在每个后继时隙中以概率p重传帧，知道帧被无碰撞传输出去

优点：

  • 单个活跃节点能连续地以信道全速传输
  • 高度分散：时隙需要同步，但何时重传独立
  • 简单

缺点：

  • 碰撞时浪费时隙
  • 有空闲时隙
  • 需要时钟同步
效率

效率是当有许多节点，每个都有许多帧需要发送时，成功时隙与总时隙的比值。

  • 假定N个节点，每个节点有许多帧要发送，每个时隙以概率p发送
  • 节点在一个时隙中成功发送的概率为p(1-p)N-1
  • 任何节点成功发送的概率为Np(1-p)N-1
  • 对N节点，求p使得N趋于无穷时，p的值
  • 此时极限值为1/e=0.37
纯ALOHA
  • 更简单，无同步要求
  • 当帧首个到达，立即传输
  • 如果发生碰撞，则以概率p重传，或等待一个帧的传输时间
  • 碰撞概率增加：t0发送的帧可能会与[t0-1, t0+1]发送的其他帧碰撞

由于碰撞概率增加，纯ALOHA的效率仅为时隙ALOHA效率的一半

CSMA（Carrier Sensing Multiple Access）

CSMA在传输前侦听：

  • 如果侦听到信道空闲：传输整个帧
  • 如果侦听到信道忙：推迟传输
CSMA碰撞

传输延迟意味着两个节点也许不能听到其他节点传输。发送碰撞意味着整个分组传输时间被浪费。

距离与传播时延决定碰撞概率。传播时延越大，载波侦听不能检测到另一个节点已经开始传播的概率越大。

CSMA/CD 带碰撞检测的载波侦听多帧访问

CD: Collision Detection

CSMA/CD同样有载波侦听。与CSMA不同的是，它具有碰撞检测功能：

  • 一旦检测到碰撞，立即停止传输，减小信道浪费
  • 等待一段随机时间后，继续侦听
二进制指数后退算法
  • 当传输一个给定帧时，在经历n个碰撞后，节点从{0,1,…,2n-1}中选择一个值K，并等待K个周期（512比特时间）
传播时延对CSMA/CD的影响

如果发送方在检测到碰撞（2τ时间）之前，已经把帧发送完，发送方就会认为这一帧成功发送；因此，需要限制最小帧：L/R≥2τ (τ=dmax/V)。

“轮流”MAC协议

轮询协议

  • 主节点依次邀请从节点进行传输
  • 关注问题：
    • 轮询开销
    • 轮询延迟
    • 单点故障（主节点）

令牌协议

  • 控制令牌从一个节点顺序地传递到下一个
  • 令牌报文
  • 关注问题：
    • 令牌开销
    • 时延
    • 单点故障（令牌）

LAN

以太网

以太网是一种“占支配性的”有线LAN技术，它分为两种：

  • 共享式以太网：
    • 网络中所有主机的收发都依赖于同一套物理介质，即共享介质。
    • 同一时刻只能有一台主机在发送，各主机通过遵循CSMA/CD规则来保证网络的正常通讯
  • 交换式以太网：
    • 存储转发交换机
    • 解决了共享信道的碰撞问题
    • 工作在第二层

以太网帧结构

  • 以太网的MTU是1500字节，如果超过则被分片。
  • 最小长度是46字节（发送46字节的速度>2τ）
  • 前导码（8字节）
    • 模式为10101010的七个字节，后跟模式为10101011的一个字节
    • 用于同步接收方，发送方时钟速率
  • MAC地址字节：6字节
    • 如果适配器接收具有匹配地址的帧，则交付给网络层协议，否则丢弃
  • 类型（2字节）
    • 指示较高层协议（大多数为IP）
  • CRC（4字节）
    • 若检测到差错就丢弃（无连接，不可靠）

链路层编址，ARP

每个节点接口有网络层地址和链路层地址。网络层地址即IP地址，用于把分组送到目的网络。链路层地址为MAC地址，用于把数据帧从一个节点传送到另一个节点（同一网络中）。

MAC地址（LAN地址、物理地址）

  • 节点“网卡”本身所带的地址（唯一）
  • 长度为6字节，共248个。FF-FF-FF-FF-FF-FF
  • 用16进制表示，每个字节表示为一个16进制数
  • 网卡的MAC地址一般是永久的（生产时固化在ROM中）

ARP：地址解析协议

高层程序希望与IP地址交往，而实际通信必须使用物理地址。

地址转换由ARP协议实现。LAN上的每一个节点（主机/路由器）都有ARP表。ARP表存放对某些LAN节点的IP/MAC地址映射：<IP地址;MAC地址;TTL>，其中TTL表示地址映射将被删除的时间（通常是20分钟）。

ARP的特点是即插即用，节点创建它们的ARP表无需网络管理员的干涉。

情况一：两主机（A和B）在同一子网中

  • 假设B的MAC地址不在A的ARP表中，则ARP工作过程如下：
    • A广播ARP查询分组，包含B的IP地址
      • 目的地址=FF-FF-FF-FF
      • 在LAN上的所有机器接收ARP请求
    • B接收ARP分组，用它的MAC地址回答A
      • 帧发送到A的MAC地址（单薄）
    • A在它的ARP表中缓存（保存）IP到MAC的地址对，直到超时

情况一：两主机（A和B）不在同一子网中
只有目的路由器与目的主机直接交付ARP数据报，两主机之间不直接交换数据报。主机只获取对应子网路由器的MAC地址。

假设主机A的ARP表没有记录路由器的MAC地址

  • A发送ARP询问获得与路由器IP地址对应的MAC地址
  • A发送帧到路由器R（间接交付）
  • 路由器获取目的IP地址，查询转发表确定输出端口
  • 路由器R发送帧到主机B（直接交付）

交换机

交换机的任务是接收入链路层的帧，并将其转发到出链路。交换机对于主机和路由器来说是透明的。它的功能有两点：

  • 过滤：决定一个帧应该被转发还是丢弃
  • 转发：决定一个帧应该被导向哪个接口
交换机表

交换机的功能借助于交换机表实现，它的内容有：

  • 一个MAC地址
  • 通向该MAC地址的交换机接口
  • 表项放置在表中的时间
转发/过滤工作流程

假定目的地址为A的帧从接口x到达

  • 交换机表中没有A表项，交换机向除了x以外的所有接口广播该帧
  • 交换机表中有该表项，转发接口为x，则过滤该帧
  • 交换机表中有该表项，转发接口为y，则向接口y转发该帧

交换机表是自动、动态和自治建立的，具有自学习能力

交换机vs路由器
  • 两者都是存储转发设备
    • 路由器：网络层设备
    • 交换机：链路层设备
  • 路由器维护路由表和转发表，实现选路算法（不是即插即用）
  • 交换机维护交换机表, 实现过滤/转发、自学习算法

交换机多端口间可以同时传输
