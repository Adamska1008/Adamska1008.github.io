计算机网络笔记（四）- 网络层-数据平面

发表于 2022-08-26 更新于 2023-01-18 分类于 《计算机网络：自顶向下方法》学习笔记

计算机网络-传输层-数据平面知识点。

网络层从发送主机到接收主机传输段：

  • 在发送侧网络层将段数据封装到数据报，向相邻路由器发送
  • 在接受侧网络层，接收来自相邻路由器的数据报，向传输层交付

核心功能——转发与路由

转发（forwarding）：将分组从路由器的输入端口转移到合适的输出端口。

  • 硬件实现
  • 路由器本地动作

路由（routing）：确定分组从源到目的经过的路径（路由算法）。

  • 选路算法
  • 软件实现
  • 路由器全局动作

某些网络需要连接建立的功能（ATM、帧中继、X.25）。数据分组传输之前两端主机需要首先建立虚拟/逻辑连接。网络层连接是主机到主机的连接，网络设备参与其中，传输层连接是两个应用进程之间的连接。

数据平面和控制平面

  • 数据平面
    • 本地，每台路由器的功能
    • 确定如何将数据报从输入链路转发到输出链路
    • IP转发和通用转发
  • 控制平面
    • 网络范围内的逻辑
    • 确定数据报沿着从源主机到目的主机的端到端路径中路由器之间的路由方式
    • 两种控制平面方式
      • 传统路由算法：在路由器中实现
      • 软件定义网络：在远程服务器中实现

路由表主要包含三个元素：目标地址，掩码，下一条，它由路由算法生成。通过路由表生成转发表。

网络层服务模型
Internet网络服务模型：尽力而为，不保证带宽、丢包、顺序、定时、拥塞控制。

无连接服务：

  • 无需先建立连接
  • 每个分组独立确定传输路径
    • 不同分组（同源、同目的）可能传输路径不同
  • 代表：数据报网络（datagram network）

连接服务：

  • 首先确定从源到目的经过的路径（建立连接）
  • 沿该路径建立分组，系列分组传输路径相同
  • 传输结束后拆除链接
  • 代表：虚电路网络（virtual-circuit）

路由器

路由器由以下部分组成：

  • 输入/输出端口
  • 交换结构
  • 选路处理器

路由器的主要功能：

  • 运行路由算法/与远程控制器通信得到转发表
  • 根据转发表对IP分组进行转发
  • 提供多种网络类型接口，完成不同网络的互联

输入端口处理

当路由器的输入端口从线路接收分组时，处理过程如下：

  • 物理层：将一条输入的物理链路与路由器相连
  • 数据链路层：剥去帧首部尾部，送到网络层队列中等待处理
  • 网络层：处理排队分组，查找转发表决定输出端口
    • 输入端口有转发表的副本
    • 转发有两种：
      • 传统转发：基于首部目的IP地址转发
      • 通用转发：基于一系列首部值转发

基于目的地址转发

每台路由器有一个将目的地址映射到链路接口的转发表，使用最长匹配算法。
例如：

前缀匹配 链路接口

11001000 00010111 00011000 1
11001000 00010111 00011 2
other 3

目的地址11001000 00010111 00010110 10100001匹配0，目的地址
11001000 00010111 00011000 10101010可以匹配1和2，于是匹配最长的1。

交换结构

  • 经内存交换：输入端口→总线→内存→总线→输出端口。不能同时转发两个分组。
  • 经总线交换：经一个共享主线连接。存在总线竞争，只能连通一个输入和输出端口。
  • 经互联网络交换：由2N条总线组成互联网络，连接N个输入端口和输出端口。

端口排队

在输入端口，当交换结构比输入端口处理慢时，出现输入端口排队延迟。此时可能发生溢出丢包；除此之外，一次只能向一个输出端口转发，该队列前的数据包也会阻挡后面的数据包。

输出端口收到转发数据包时同样有缓存处理，会分组排队。此时同样可能发生溢出丢包。

调度机制

FIFO

先进先出，以到达队列的顺序发送。

丢弃策略：

  • 丢弃刚到达的分组
  • 丢弃优先级低的分组
  • 随机丢弃

优先级

发送优先级最高的分组。同一优先级按FIFO处理。基于标识/头部信息分类。

循环排队RR

分类无优先级，循环检查队列，为每个队列一次发送一个分组。

加权公平排队

RR+优先级，轮换处理分组，但优先级高的队伍处理分组数量多。

IPv4协议

IP数据报结构

首部+数据（TCP段、UDP段）

IP地址固定首部20字节，与TCP相同。

首部：

  • 固定部分
    • 版本号：4bit，4 or 6
    • 首部长度：4bit，IP分组首部长度，以四字节为单位，如5表示20bytes
    • 服务类型：8bit，指示期望获得哪种类型的服务，只有在网络提供区分服务时才有效，一般情况下不使用
    • 总长度：16bit，表示IP分组的总字节数，最大IP分组总长度65535字节。
    • 标识、标志位、片偏移（IP分片部分介绍）。
    • 生存时间（TTL）：IP分组可以通过的路由器数（跳步数）。
      • 路由器转发一次分组，TTL减1，如果TTL=0，路由器则丢弃该分组。
    • 协议：指示IP分组封装的是哪个协议的数据包。
    • 首部校验和字段：16bit，实现对IP分组的差错检测。采用反码算数运算求和。
    • 源、目的IP地址：各占32bit
  • 可变部分
    • 选项字段，范围0~40B：携带安全、源选路径、时间戳、路由记录等内容。实际上很少使用。
    • 填充字段：保证首部字长是4字节的倍数。

IP分片

最大传输单元（MTU）：链路层数据帧可封装数据的上限。

大IP数据报向较小MTU链路转发时，可以被分片（fragmented）。路由器只负责分片，目的主机负责重新装配。

IP首部（标识、标志位、片偏移）用于标识分片以及确定分片的相对顺序。

标识字段占16位，标识一个IP分组。所有分片的数据报有相同标识。

标志位字段占3位：保留，DF(Don’t Fragment)，MF(More Fragment)。

  • DF=1：禁止分片
    DF=0：允许分片
  • MF=1：已被分片，不是最后一片
    MF=0：是最后一片，或未被分片

片偏移：一个IP分组分片封装原IP分组数据的相对偏移量。以8字节为单位。

IP分片过程：
假设原IP分组总长度为L，待转发链路的MTU为M。若L>M，且DF=0，则可以/需要分片。

分片时每个分片的标识复制原分组的标识。分片时，除最后一个分片，其他分片均分为MTU允许的最大分片。

  • 一个最大分片可封装的数据应该是8字节的倍数，一个最大分片可封装的数据为 d=⌊M−208⌋×8d=\lfloor\cfrac{M-20}{8} \rfloor\times 8d=⌊8M−20​⌋×8
  • 需要的总片数为 ⌈L−20d⌉\lceil\cfrac{L-20}{d}\rceil⌈dL−20​⌉
  • 有效载荷是8字节的倍数，片偏移字段取值为 Fi=d8×(i−1)F_i=\cfrac{d}{8}\times(i-1)Fi​=8d​×(i−1)

IPv4编址

使用IP地址（32bit，IPv4）编号标识主机、路由器的接口。

两级IP地址：高比特部分称为网络号，低比特部分称为主机号。具有相同网络号的集合构成IP子网（Subnets），子网中的端口网络号相同，主机号不同。

IP子网还有一个特点：不跨越路由器可以彼此物理联通。

子网的表示方法：前缀（网络号）/长度（子网掩码），例如223.1.1.0/24，表示前24位为子网号部分。路由器仅根据网络号转发分组。

子网掩码

子网掩码形如IP地址，其中网络位全部取1，主机位全部取零。使用子网掩码即可辨别是否使用子网划分。

子网掩码还用于确定应该将分组转发到哪个子网。将目的IP地址与子网掩码按位与运算即可得到子网地址。

CIDR与路由聚合

无类域间路由（CIDR:Classless InterDomain Routing） 消除传统的A类、B类、C类地址界限，网络前缀可以为任何长度。融合子网地址与子网掩码，方便子网划分。

格式：a.b.c.d/x，其中前半部分为点分式IP地址，x为前缀长度。

优点：

  • 提高IPv4地址空间分配效率
  • 提高路由效率
    • 将多个子网聚合为一个较大的子网
    • 构造超网（supernetting）

层次划分子网

ISP在获得一段连续地址块后，可以将一个块分成几个部分供多个内部网络使用，但对外部世界仍然像单个网络一样。

练习

某单位申请了一段IP地址：200.23.16.0/23，单位内由四个部门A、B、C、D组成，每个部门的主机数量分别为：40（A）、200（B）、100（C）、50（D）。试将总地址块划分成4个子网。

  1. 写出每个子网
  2. 每个子网的网络前缀、子网掩码
  3. 每个子网的广播地址，主机可用地址范围

200.23.16.0/23的二进制表示为11001000 00010111 00010000 00000000。首先为B分配256个IP地址200.23.16.0/24，再为C分配128个IP地址200.23.17.0/25，最后将剩下的地址平分给A与D。

DHCP

主机可以通过静态配置获得主机的IP地址。默认网关：路由器接入子网的IP地址。子网中的主机通过默认网关传输数据到路由器。

除了静态获取之外，还可以动态配置。动态主机配置协议（Dynamic Host Configuration Protocol） 从服务器动态获取**IP地址、子网掩码、默认网关地址、DNS服务器名称与IP地址 **。“即插即用”并支持地址重用。当主机不进行网络通信时，可以归还地址。支持移动用户加入网络。

协议内容

  • 主机广播“DHCP discover”（发现报文）
  • DHCP服务器利用“DHCP offer”（提供报文）进行响应
  • 主机请求IP地址（广播）：“DHCP request”（请求报文）
  • DHCP服务器分配IP地址：“DHCP ack”（确认报文）

应用层实现：

  • 封装到UDP数据报
  • IP广播
  • 链路层广播

网络地址转换（NAT）

本地网络使用私有IP，对外界网络，所有主机使用相同IP地址，用端口号对不同主机加以区分。

优势：

  • 只需/能从ISP申请一个IP地址
  • 本地网络设备IP地址的变更无需通告外界网络
  • 变更ISP时，无需修改内部网络设备IP地址
  • 内部网络设备对外界网络不可见，不可直接寻址（安全）

实现：

  • 替换：利用（NAT IP地址，新端口号）替换每个外出IP数据报的（源IP地址，源端口号）
  • 记录；将没对替换信息存储到NAT转换表中
  • 替换：利用转换表，将外网信息修正为内网信息

争议：

  • 处理了第三层以上的内容
  • 违背端到端通信原则，使得应用开发者必须考虑到NAT的存在
  • 地址短缺问题应该由IPv6来解决

NAT网络地址转换

IPv4的地址数面临着用尽的现状。通过NAT，提供内部私有地址与公网地址的转换，使得一个子网能支持更多的主机，支持内网和公网的通信。

所有离开本地网络的数据报拥有相同的单一源NAT IP地址，不同的源端口号。

向公网发送数据报时，NAT路由器改变数据报的地址，为每一个局域网地址分配一个公网ip下的端口号，并在接收回答时进行转发。

优点：

  1. 解决IPv4地址耗尽的问题。
  2. 本地网络设备ip地址的变更无需通告外界网络。
  3. 变更ISP时无需修改内部ip地址。
  4. 内部网络设备对外部不可见，不能直接寻址，安全。

缺点：

  1. 路由器应该只处理第三层的功能。
  2. 违反了端到端原则，例如p2p应用的设计者必须考虑NAT
  3. 地址短缺应当由IPv6解决。

互联网控制报文协议ICMP

ICMP支持主机或路由器

  • 差错（或异常）报告
  • 网络查询

两类报文：

  • 差错报告报文（5种）
    • 目的不可达
    • 源抑制-拥塞控制（Source Quench）（缓存已满）
    • 超时/超期
    • 参数问题
    • 重定向（Redirect）
  • 网络探询报文（2组）
    • 回声（ping）请求与应答报文（Reply）
    • 时间戳请求与应答报文

不发送差错报文的情况：

  • 对ICMP差错报文不再发送ICMP差错报告报文
  • 除第一个IP数据分片以外，对后续分片均不发送ICMP差错报告报文
  • 对多播IP数据报不发送
  • 对特殊地址（127.0.0.0，0.0.0.0）不再发送

格式：
ICMP报文封装到IP数据报中传输（数据部分）：

  • 类型（8bit）、代码（8bit）、检验和（16bit）
  • 四字节取决于类型
  • 数据部分（长度取决于类型）

Traceroute

源向目的地发送一系列具有不可到UDP端口的IP数据报，当第n个数据包到达第n个路由器，路由器丢弃数据包并向源发送一个ICMP报文（TTL过期），报文包括路由器名字和IP地址。

当ICMP到达，源计算RTT。Traceroute重复3次。

IPv6

初始动机：32位地址将被分配完
附加动机：

  • 首部格式提升处理/转发速率
  • 首部变化以促进QoS

格式：

  • 固定长度40字节的基本首部
  • 不允许分片

  1. 版本、流量类型、流标号
    • 流量类型：标识数据的优先级
    • 流标签：标识同一“流”中的数据报
  2. 有效载荷长度、下一首部、跳数限制
  3. 源地址
  4. 目标地址

与IPv4的区别：

  1. 去除检查和：减小每跳的处理时间
  2. 分片和重组：不允许路由器分片
  3. 选项：移到首部之外，由“下一个首部”字段指示
  4. 添加新版本的ICMP

IPv6地址的一般格式：冒号16进制。

从IPv4迁移到IPv6：

  1. 双栈：结点具备发送两种协议的功能
  2. 隧道：在IPv4路由器之间IPv6数据报作为IPv4数据报的负载。

OpenFlow数据平面抽象

路由器中的流表（由远程控制器计算和发布）定义了路由器匹配+动作规则。

源 目的 动作 计数器
1.2.*.* 3.4.5.* drop 2
*.*.*.* 3.4.*.* forward(2) 3
