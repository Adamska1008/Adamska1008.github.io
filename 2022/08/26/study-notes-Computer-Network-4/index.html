计算机网络笔记（四）- 网络层-数据平面

发表于 2022-08-26 更新于 2022-12-27 分类于 《计算机网络：自顶向下方法》学习笔记

计算机网络-传输层-数据平面知识点

  • 从发送主机到接收主机传输段：
    • 在发送侧网络层将段数据封装到数据报，向相邻路由器发送
    • 在接受侧网络层，接收来自相邻路由器的数据报，向传输层交付

核心功能——转发与路由

转发（forwaring）：将分组从路由器的输入端口转移到合适的输出端口。

  • 硬件实现
  • 路由器本地动作

路由（routing）：确定分组从源到目的经过的路径（路由算法）。

  • 选路算法
  • 软件实现

某些网络需要连接建立的功能（ATM、帧中继、X.25）。数据分组传输之前两端主机需要首先建立虚拟/逻辑连接。网络层连接是主机到主机的连接，网络设备参与其中，传输层连接是两个应用进程之间的连接。

数据平面和控制平面

  • 数据平面
    • 本地，每台路由器的功能
    • 确定如何将数据报从输入链路转发到输出链路
    • IP转发和通用转发
  • 控制平面
    • 网络范围内的逻辑
    • 确定数据报沿着从源主机到目的主机的端到端路径中路由器之间的路由方式
    • 两种控制平面方式
      • 传统路由算法：在路由器中实现
      • 软件定义网络：在远程服务器中实现

网络层服务模型
Internet网络服务模型，不保证带宽、丢包、顺序、定时、拥塞控制。

无连接服务：

  • 不事先为系列分组的传输确定传输路径
  • 每个分组独立确定传输路径
  • 不同分组（同源、同目的）可能传输路径不同
  • 代表：数据报网络（datagram network）

连接服务：

  • 首先为系列分组的传输确定从源到目的经过的路径（建立连接）
  • 沿该路径建立分组，系列分组传输路径相同
  • 传输结束后拆除链接
  • 代表：虚电路网络（virtual-circuit）

虚电路网络（Virtual circuits）

虚电路网络提供网络层连接服务，实现主要在网络核心而非两端。

虚电路：一条从源主机到目的主机，类似于电路的路径（逻辑连接）。

  • 分组交换
  • 每个分组的传输利用链路的全部带宽
  • 源到目的路径经过的网络层设备共同完成虚电路功能

通信过程：

  • 呼叫建立→数据传输→拆除呼叫。
  • 每个分组携带虚电路标识（VCID）而不是目的主机地址。
  • 虚电路经过的每个网络设备，维护每条经过它的虚电路连接状态。
  • 链路、网络设备资源可能面向VC进行预分配

VC 的具体实现

每条虚电路包括

  1. 从源到目的主机的一条路径
  2. 虚电路号（VCID），沿路每段链路一个编号
  3. 沿路每个网络层设备（如路由器），利用转发表记录经过的每条虚电路

同一条虚电路，在每一段链路上的VCID通常不同。路由器转发分组时依据转发表改写/替换虚电路号。

转发表主要记录以下内容：

  • 输入接口
  • 输入VC
  • 输出接口
  • 输出VC

虚电路的建立对应增加一条转发表项。拆除对应删除一条转发表项。

虚电路信令协议

用于VC的建立、维护和拆除（路径选择），应用于虚电路网络。

数据报网络

数据报网络是无连接服务。每个分组携带目的地址。路由器根据分组的目的地址转发分组，每个分组独立选路。

转发表记录携带的地址与对应的输出链路，根据地址确定输出链路。多数情况下，转发表不针对某个具体地址进行转发，而是根据地址范围进行转发。具有共同地址范围的项使用相同的输出链路。

转发表可能有多条接口可以匹配。在检索转发表时，优先选择与分组目的地址匹配前缀最长的入口。

路由器

路由器由以下部分组成：

  • 输入/输出端口
  • 交换结构
  • 选路处理器

路由器的主要功能：

  • 运行路由算法/与远程控制器通信得到转发表
  • 根据转发表对IP分组进行转发
  • 提供多种网络类型接口，完成不同网络的互联

基于目的地址转发
每台路由器有一个将目的地址映射到链路接口的转发表，使用最长匹配算法。

IPv4协议

IP数据报结构

首部+数据（TCP段、UDP段）

首部：

  • 固定部分
    • 版本号：2bit，4 or 6
    • 首部长度：4bit，IP分组首部长度，以四个字节为单位，如5表示20bytes
    • 服务类型：8bit，指示期望获得哪种类型的服务，只有在网络提供区分服务时才有效，一般情况下不使用
    • 总长度：16bit，表示IP分组的总字节数，最大IP分组总长度65535B。
    • 标识、标志位、片偏移
    • 生存时间（TTL）：标识IP分组在网络中可以通过的路由器数（跳步数）。路由器转发一次分组，TTL减1，如果TTL=0，路由器则丢弃该分组。
    • 协议：指示IP分组封装的是哪个协议的数据包。
    • 首部校验和字段：16bit，实现对IP分组的差错检测。采用反码算数运算求和。
    • 源、目的IP地址：各占32bit
  • 可变部分
    • 选项字段，范围0~40B：携带安全、源选路径、时间戳、路由记录等内容。实际上很少使用。
    • 填充字段：保证首部字长是4字节的倍数。

IP分片

最大传输单元（MTU）：链路层数据帧可封装数据的上限。

大IP分组向较小MTU链路转发时，可以被分片（fragmented）。路由器只负责分片，目的主机负责重组。

IP首部的相关字段用于标识分片以及确定分片的相对顺序：总长度、标识、标志位、片偏移。

标识字段占16位，标识一个IP分组。

标志位字段占3位：保留，DF(Don’t Fragment)，MF(More Fragment)。

  • DF=1：禁止分片
    DF=0：允许分片
  • MF=1：已被分片，不是最后一片
    MF=0：是最后一片，或未被分片

片偏移：一个IP分组分片封装原IP分组数据的相对偏移量。以8字节为单位。

IP分片过程：
假设原IP分组总长度为L，待转发链路的MTU为M。若L>M，且DF=0，则可以/需要分片。

分片时每个分片的标识复制原分组的标识。分片时，除最后一个分片，其他分片均分为MTU允许的最大分片。

  • 一个最大分片可封装的数据应该是8字节的倍数，一个最大分片可封装的数据为 d=⌊M−208⌋×8d=\lfloor\cfrac{M-20}{8} \rfloor\times 8d=⌊8M−20​⌋×8
  • 需要的总片数为 ⌈L−20d⌉\lceil\cfrac{L-20}{d}\rceil⌈dL−20​⌉

IP编址

需要编制的内容：实现网络层功能的接口。使用IP地址（32bit，IPv4）编号标识主机、路由器的接口。

IP地址高比特部分称为网络号，低比特部分称为主机号。相同网络端口IP的网络号是相同的，主机号是不同的。具有相同网络号的集合构成IP子网（Subnets）。IP子网还有一个特点：不跨越路由器可以彼此物理联通。

有类IP地址

有类编址采用二分法。

定义50%的IP地址为A类地址，规定第一个8位组为网络号，最高位比特固定是0，后24位是主机号。0.0.0.0~127.255.255.255

剩余50%中，最高比特位为1，次高比特位为0的是B类定制，占剩余的50%，前16bit为网络号，后16比特为主机号。128.0.0.0~191.255.255.255

再剩余的50%中，前三位为110，前24位为网络号，后8位为主机号。192.0.0~223.255.255.255。

继续二分D类地址，不再区分网络号与主机号，前四位为1110。剩余为E类地址，前四位为1111。D类地址为多播地址。E类地址保留，作为研究使用。

A、B、C类地址可以用于标记互联网中的端口。

特殊地址：

NetID HostID 作为IP分组源地址 作为IP分组目的地址 用途
全0 全0 可以 不可以 在本网范围内表示本机
全0 特定值 不可以 可以 本网内某个特定主机
全1 全1 不可以 可以 本网广播地址（路由器不转发）
特定值 全0 不可以 不可以 网络地址，表示一个网络
特定值 全1 不可以 可以 直接广播地址，对特定网络上所有主机进行广播
127 非全0非全1 可以 可以 用于本地软件环回测试，称为环回地址

私有IP地址

在公共互联网上无效，只能在内部网络使用。

Class NetIDs Blocks
A 10 1
B 172.16 to 172.31 16
C 192.168.0 to 192.168.255 256

IP子网划分与子网掩码

子网号（SubID）是原网络主机号部分比特。将通信流量进行有效隔离。

子网掩码形如IP地址，其中NetID、SubID位全部取1，HostID位全部取零。使用子网掩码即可辨别是否使用子网划分。

子网掩码还用于确定应该将分组转发到哪个子网。将目的IP地址与子网掩码按位与运算即可得到子网地址。

CIDR与路由聚合

无类域间路由（CIDR:Classless InterDomain Routing） 消除传统的A类、B类、C类地址界限，网络前缀可以为任何长度。融合子网地址与子网掩码，方便子网划分。

格式：a.b.c.d/x，其中前半部分为点分式IP地址，x为前缀长度。

优点：

  • 提高IPv4地址空间分配效率
  • 提高路由效率
    • 将多个子网聚合为一个较大的子网
    • 构造超网（supernetting）

DHCP

主机可以通过静态配置获得主机的IP地址。默认网关：路由器接入子网的IP地址。子网中的主机通过默认网关传输数据到路由器。

除了静态获取之外，还可以动态配置。动态主机配置协议（Dynamic Host Configuration Protocol） 从服务器动态获取IP地址、子网掩码、默认网关地址、DNS服务器名称。“即插即用”并支持地址重用。当主机不进行网络通信时，可以归还地址。支持移动用户加入网络。

协议内容

  • 主机广播“DHCP discover”（发现报文）
  • DHCP服务器利用“DHCP offer”（提供报文）进行响应
  • 主机请求IP地址（广播）：“DHCP request”（请求报文）
  • DHCP服务器分配IP地址：“DHCP ack”（确认报文）

应用层实现：

  • 封装到UDP数据报
  • IP广播
  • 链路层广播

网络地址转换（NAT）

本地网络使用私有IP，对外界网络，所有主机使用相同IP地址，用端口号对不同主机加以区分。

优势：

  • 只需/能从ISP申请一个IP地址
  • 本地网络设备IP地址的变更无需通告外界网络
  • 变更ISP时，无需修改内部网络设备IP地址
  • 内部网络设备对外界网络不可见，不可直接寻址（安全）

实现：

  • 替换：利用（NAT IP地址，新端口号）替换每个外出IP数据报的（源IP地址，源端口号）
  • 记录；将没对替换信息存储到NAT转换表中
  • 替换：利用转换表，将外网信息修正为内网信息

争议：

  • 处理了第三层以上的内容
  • 违背端到端通信原则，使得应用开发者必须考虑到NAT的存在
  • 地址短缺问题应该由IPv6来解决

NAT网络地址转换

外部主机可能希望主动访问内部主机。NAT技术提供内部私有地址与公有地址的转换，支持内网与公网的通信。

所有离开本地网络的数据报拥有相同的单一源NAT IP地址，不同的源端口号。NAT路由器改变数据报的地址，为每一个局域网地址分配一个公网ip下的端口号，并在发出时进行修改，接收时进行转发。

优点：

  1. 本地网络设备ip地址的变更无需通告外界网络。
  2. 变更ISP时无需修改内部ip地址。
  3. 内部网络设备对外部不可见，不能直接寻址，安全。

缺点：

  1. 路由器应该只处理第三层的功能。
  2. 违反了端到端原则，例如p2p应用的设计者必须考虑NAT
  3. 地址短缺应当由IPv6解决。

互联网控制报文协议ICMP

ICMP支持主机或路由器

  • 差错（或异常）报告
  • 网络查询

两类报文：

  • 差错报告报文（5种）
    • 目的不可达
    • 源抑制-拥塞控制（Source Quench）（缓存已满）
    • 超时/超期
    • 参数问题
    • 重定向（Redirect）
  • 网络探询报文（2组）
    • 回声（ping）请求与应答报文（Reply）
    • 时间戳请求与应答报文

不发送差错报文的情况：

  • 对ICMP差错报文不再发送ICMP差错报告报文
  • 除第一个IP数据分片以外，对后续分片均不发送ICMP差错报告报文
  • 对多播IP数据报不发送
  • 对特殊地址（127.0.0.0，0.0.0.0）不再发送

格式：
ICMP报文封装到IP数据报中传输（数据部分）：

  • 类型（8bit）、代码（8bit）、检验和（16bit）
  • 四字节取决于类型
  • 数据部分（长度取决于类型）

Traceroute

源向目的地发送一系列具有不可到UDP端口的IP数据报，当第n个数据包到达第n个路由器，路由器丢弃数据包并向源发送一个ICMP报文（TTL过期），报文包括路由器名字和IP地址。

当ICMP到达，源计算RTT。Traceroute重复3次。

IPv6

格式：

  • 固定长度40字节的基本首部
  • 不允许分片

  1. 版本、流量类型、流标号
    • 流量类型：标识数据的优先级
    • 流标签：标识同一“流”中的数据报
  2. 有效载荷长度、下一首部、跳数限制
  3. 源地址
  4. 目标地址

与IPv4的区别：

  1. 去除检查和：减小每跳的处理时间
  2. 分片和重组：不允许路由器分片
  3. 选项：移到首部之外
  4. 添加新版本的ICMP

从IPv4迁移到IPv6：

  1. 双栈：结点具备发送两种协议的功能
  2. 隧道：在IPv4路由器之间IPv6数据报作为IPv4数据报的负载。
