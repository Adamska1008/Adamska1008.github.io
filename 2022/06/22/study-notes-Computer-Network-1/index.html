计算机网络笔记（一）-计算机网络概述

发表于 2022-06-22 更新于 2022-10-26 分类于 《计算机网络：自顶向下方法》学习笔记

内容：计算机网络的定义、结构、数据交换过程以及体系模型。

定义

计算机网络是互联的、自治的计算机集合

  • 自治-无主从关系
  • 互联-互联互通

因特网组成

  • 主机（host）或端系统（end systm）：数据处理，运行网络程序
    • Server, Mobile, Router, Modem, Base Station, Smart phone
  • 通信链路（communication link）：把端系统连接到一起的物理线路
    • 同轴电缆、双绞线、光纤、无线
    • 多种速率（bit/s bps）
  • 分组交换机（packet switch）：中间交换设备
    • 功能：接收、转发分组
    • 类型
      • 路由器（router）
      • 链路层交换机（link-layer switch）

分组交换
发送端将要发送的数据分成若干较小的块，添加首部形成分组（packet），分别发送到目的端，再组装成原数据。

因特网服务

端系统通过因特网服务商（Internet Service Provider, ISP） 接入互联网。

从服务角度，因特网是为应用程序提供服务的基础设施，包括电子邮件，Web，社交网络，流媒体。这些应用程序被称为分布式应用程序（distributed application）。

与因特网相连的端系统提供了一个应用程序接口（Application Programming Interface, API），该API规定了运行在一个端系统上的软件请求因特网基础设施向运行在另一个端系统上的特定目的地软件交付数据的方式。

低层次的ISP通过国家、国际的高层ISP互联，实现世界范围的通信。每个ISP独立管理，运行ISP协议。

协议

网络协议是为进行网络中的数据交换而建立的规则、标准或约定。

协议定义了在两个或多个通信实体之间交换的报文格式和次序，以及报文发送和/或接收一条报文或事件所采取的动作。

协议的三要素：

  • 语法（Syntax）
    • 数据与控制信息的结构或格式
    • 信号电平
  • 语义（Semantics）
    • 需要发出何种控制信息
    • 完成何种动作以及做出何种响应
    • 差错控制
  • 时序（Timing）
    • 事件顺序
    • 速度匹配

结构

由三部分组成

  • 网络边缘：
    • 应用程序
    • 主机、端系统
  • 接入网络
    • 有线/无线通信介质
  • 网络核心
    • 路由器（分组转发设备）

网络边缘

  • 客户/服务器（client/server）应用模型。客户发送请求，接收服务器响应。
  • 对等（peer-peer, P2P）应用模型。无（或不依赖）专用服务器，通信在对等实体上直接进行。

接入网络

将端系统连接到其边缘路由器的物理链路，可分为住宅接入网络、机构接入网络和移动接入网络。

边缘路由器：端系统到远程端系统路径上的第一台路由器。

家庭接入

典型的家庭网络使用调制解调器（下文两种方式之一）连接到中心局或头端。内部部署路由器实现地址转换。

调制：将数字信号转化成模拟信号。
解调：将模拟信号转化为数字信号。

数字用户线路（Digital Subscriber Line, DSL）

利用家庭已有的电话线，通过电话线实现数据传输。利用多路复用技术，将话音信号和数据信号分离，数据通信接入Internet，语音通信接入电话网。

  • 50kHz~1MHz：下行数据
  • 4~50kHz：上行数据
  • 0~4kHz：电话
电缆网络/混合光纤同轴电缆(HFC)

利用有线电视网络。使用频分多路复用，在不同频带上传输不同频道。有时被称为HFC（混合光纤同轴电缆），速度同样为不对称的。

企业接入

通常使用局域网（LAN）连接段用户和边缘路由器。

以太网（Ethernet）

常用于机构接入网络。以太网是一种典型的有线局域网技术。连接到路由器，再通过其连接到ISP。

无线接入网络

通过基站共享无线接入网络，例如IEEE802.11无线局域网。

  • 无线局域网（LANs）：同一建筑内，覆盖范围约几十米。
  • 广域无线接入：通过电信运营商，接入范围在几十公里

物理媒体

分为两类：导引型媒体、非导引型媒体

  • 非导引型媒体
    • 陆地无线电信道
    • 卫星无线电信道
      • 同步卫星
      • 近地轨道卫星

网络核心

关键功能：路由+转发

路由：确定分组从源到目的传输路径。通过路由算法+本地转发表实现。

转发：将分组从路由器的输入端口交换至正确的输出端口

Internet结构：网络的网络

端系统通过接入ISP（access ISPs）连接到Internet，例如家庭，公司，大学ISP，而接入ISP自身必须进一步互联。直接互联的成本太高。如果建立全球性的全球承载ISP，承载所有接入ISP，则接入量太大，且不符合实际需求。这个模型下，接入ISP是客户，承载ISP是提供商。由于竞争的存在，全球承载ISP可能有多个。

事实上，在一个给定区域，可能有一个区域ISP（reginal ISP），例如国家。这些ISP与全球承载ISP相连。在其下又有区域ISP，例如省级ISP，接入接入ISP。

为使这些ISP互联，可以使用：

  • 对等链路（peering link），使ISP互相连接
  • 第三方的互联网交换节点（Internet exchange point），连接全球承载ISP和区域ISP

还有一些网络供应商，如谷歌，可能构建属于自己的ISP。

数据交换

数据交换是网络核心解决的主要问题，数据通过网络核心从源主机到达目的主机。

如果没有数据交换，任意两条主机之间连一条物理链路，则需要 O(n2)O(n^2)O(n2) 条物理链路。这并不可行。通过交换设备，让主机与交换设备连接，每个主机只需要一条链路。

交换设备连接到一起，就构成了交换网络。

交换设备需要两个功能：

  • 动态转接，建立端口之间的连接，通常可以并行地进行。
  • 动态分配传输资源

数据交换大致可以分为：电路交换、报文交换、分组交换

电路交换

最典型的电路交换网络是电话网络。

电路交换网络具有三个阶段：

  • 建立连接（呼叫/电路建立）
  • 通信
  • 释放资源

电路交换的特性是独占资源，双方建立的通信不能由第三方使用。

使用电路交换时，一条链路可能被几个通信共享。要共享这些中继线，使用多路复用（multiplexing） 技术。通信通过复用器进行复用后，在共享信道上传输，最后经过分用器分解。

该技术将链路/网络资源划分为“资源片”，将资源片分配给各路呼叫，每路呼叫独占分配到的资源片，资源片可能闲置。

常见的复用技术有：

  • 频分多路复用（frequency division muliplexing, FDM），用户占用不同的频率传递数据。
    • 按频率划分若干频段，每个连接专用一个频段
    • 通常具有4kHz的宽度，该宽度称为带宽
  • 时分多路复用（time division multiplexing, TDM），
    • 将时间划分为等长的时分复用帧（TDM）帧，将每个帧又划分为固定数量的时隙。
    • 每个用户在每个TDM帧中占用固定需要的时隙。通过该时隙传输数据。

缺陷

  • 效率较低：静默期专用电路空闲，网络资源被浪费
  • 创建端到端电路以及预留带宽比较复杂

报文、分组交换

报文（message）：需要传输的数据。

分组（packet）：报文分拆出来的一系列相对较小的数据包，加上头部信息。分组交换需要报文的拆分和重组，产生额外开销。分组交换按需共享链路。

工作过程：

  • 源将报文划分为分组
  • 每个分组通过网络传输
  • 目的端恢复原报文

分组以链路的最大传输速率传输，使用存储转发机制。

存储转发机制：交换机能够开始向输出链路传输该分组第一个比特时，必须接收到整个分组。

传输延迟问题

假设源主机将报文拆分为 LLL bits 的分组，在传输速率为 RRR 的链路上传送分组时，称从发送第一个bit开始，到最后一个bit结束的事件，为传输延迟。显然 delay=LRdelay = \cfrac{L}{R}delay=RL​。对于一个具有 NNN 条速率为 RRR 的链路组成的路径，端到端延迟为 NLRN\cfrac{L}{R}NRL​。

当大小为 MMM 的报文发送时，每次必须传送完整的报文到交换机，才能进行下一次传送，每个报文需要 MR\frac{M}{R}RM​ 秒。而每一个分组，则等待时间较少。

假设有两路交换机，M=7.5MB，L=1.5KB，R=1.5MB/sM = 7.5MB，L=1.5KB，R=1.5MB/sM=7.5MB，L=1.5KB，R=1.5MB/s，有 MR=5\frac{M}{R}=5RM​=5，报文交换需要三次传输，需要15秒，路由器本身需要至少7.5MB缓存。

按照分组交换，一个报文需要 1ms 传输，传输完毕后，可以同时继续向第一个路由器传输报文，同时可以向第二个路由器发送报文。经过 3ms，第一个报文传送完毕。当 4999ms 时，开始发送最后一个报文，并在 5002ms 时传输完成。此时，需要5.002s，明显快于报文交换。这是由于，分组交换并行地利用了带宽。同时，分组交换中，路由器只需要1.5KB的缓存就能工作，远小于报文交换。

分组交换中，假设有 nnn 个路由器，则交付时间为：T=MR+nLRT=\cfrac{M}{R}+\cfrac{nL}{R}T=RM​+RnL​。

两者比较：

  • 电路交换效率不高
  • 分组交换不适合实时服务：端到端时延不确定
  • 分组交换带宽贡献好
  • 分组交换利用率高

性能指标

延迟/时延

当分组等待正在传输的分组，或排队时，会产生延迟。当分组缓存已满，但仍有数据包发送时，数据可能会被丢弃，产生丢包。

主要有四种时间延迟：

  1. 结点处理延迟（nodal processing delay）：差错检测、确定输出链路。时间相对小。
  2. 排队延迟（queueing delay）：等待输出链路可用。取决于路由器拥塞程度。
  3. 传输延迟（transmission delay）：传输数据本身的延迟。取决于分组长度和链路带宽。
  4. 传播延迟（propagation delay）：信号在链路上传输的时间，与物理链路长度和速度有关。

令 aaa 表示分组到达队列的平均速率（分组/秒），比率 La/RLa/RLa/R 被称为流量强度，如果其大于 111，则队列将无限增加，所以设计系统时流量强度不能大于 111。

端到端时延

假设网络无拥塞时，有 N−1N-1N−1 台路由器，则延时累加得到端到端时延：

dend_end=N(dproc+dtrans+dprop)d_{end\_ end}=N(d_{proc}+d_{trans}+d_{prop}) dend_end​=N(dproc​+dtrans​+dprop​)

时延带宽积

时延带宽积 = 传播时延(dprop) × 带宽®，又称为以比特为单位的链路长度，可以理解为链路上能够容纳多少比特。

丢包

路由器队列缓存有限，分组到达已满队列将被丢弃，丢弃分组可能由前序节点或源重发（或不重发）。用丢包率衡量。

丢包率=丢包数已发分组总数丢包率=\cfrac{丢包数}{已发分组总数} 丢包率=已发分组总数丢包数​

吞吐量/率

表示在发送端与接收端之间传送数据吞吐量

  • 即时吞吐量
  • 平均吞吐量

假设由两条链路串联组成一条链路，吞吐量由较小的链路决定。

假设共享链路，则还需考虑共享链路，取三者最小值为实际吞吐量。

体系结构

计算机网络体系结构是分层结构。

  • 实体（entity） 表示任何可发送或接收信息的硬件或软件进程
  • 协议 是控制两个对等实体进行通信的规则的集合，协议是 “水平的”。
  • 任一层实体需要使用下层服务，遵循本层协议，实现本层功能，向上层提供服务，服务是“垂直的”。
  • 下层协议的实现对上层的服务用户是透明的。
  • 同系统的相邻层实体间通过接口进行交互，通过服务访问点SAP（Service Access Point），交换原语，指定请求的特定服务。

OSI参考模型

开放系统互连 参考模型是异构网络系统互联的国际标准，是理解网络通信的理论模型，它将网络从功能上划分成七个层次，从上到下分别是

  • 应用层
  • 表示层
  • 会话层
  • 传输层
  • 网络层
  • 数据链路层
  • 物理层

在传输网络中，主机需要完成所有七层功能。而路由器等只需要完成最下三层功能。通信基于一种传输介质。

假设主机A向B传输数据，从上到下依次处理数据，中间系统从物理层接收数据，还原到网络层，每一层数据包称之为PDU，网络层明确了如何传输，于是继续向下处理，最后在B逐层还原。在某一层看来，逻辑上相当于每一层直接通信，但实际上不是，仅仅在协议上是对等的。

前四层直接面对目的主机，称之为端到端层（end-end），后三层为非端到端。

每一层的数据称为协议数据单元，每一层向下传递时加上头部信息，而数据链路层则会额外加上尾部信息，到达物理层时，解释为比特流，在物理介质上传播。

数据分装的作用：

  • 增加控制信息
    • 构造协议数据单元（PDU）
  • 内容：
    • 地址（Address）：标识发送端和接收端
    • 差错检测编码（Error-detecting code）
    • 协议控制（protocol control）：实现附加信息，如：优先级（priority）、服务质量（QoS）、安全控制。

每层功能：

物理层

  • 接口特性：机械特性、电气特性、功能特性、规范特性
  • 比特编码
  • 数据率
  • 比特同步（时钟同步）
  • 传输模式
    • 单工
    • 半双工，例如对讲机，发收不能同时进行
    • 全双工，收发同时进行，一般使用两个信号

数据链路层

物理层解决了单一比特的传输问题，而数据链路层负责结点到结点（node-to-node） 传输，以帧作为数据传输的单位。

组帧（Framing） 将数据包装为帧，使得接收时能将数据切割为帧。

物理寻址（Physical addressing） ，在帧头中增加发送端和/或接收端的物理地址标识数据帧的发送端/接收端。当且仅当接收地址与接收端相同时，才接收。

流量控制（Flow control），匹配发送与接收速度，避免淹没接收端。

差错控制（Error control），根据差错编码判断数据是否有错。

访问控制（Access control），决定谁能使用介质，在共享链路层中有用。

网络层

负责从源主机到目的主机数据分组的交付，进行逻辑寻址，全局唯一地址，保证数据正确传送，如IP地址。

路由（Routing），路由互联网络，并分组至最终目的主机（分组转发）。

传输层

负责端到端完整报文的传输。

功能：

  • 分段和重组
  • SAP寻址
    • 确保将完整报文提交给正确进程，如端口号
  • 连接控制
  • 流量控制
  • 差错控制

会话层

会话层不分割表示层的数据，而是插入同步控制点。功能：

  • 对话控制（dialog controlling）
  • 同步（synchronization）在数据流中插入同步点

较为单薄，实际上可能不存在，下同。

表示层

处理两个系统之间信息的语法与语义（syntax and semantics）问题 。

  • 数据转化：转化成与主机无关或相关的编码
  • 加密/解密
  • 压缩/解压缩

应用层

应用之间的通信，如http、ftp、smtp，网络应用通过相应协议编写控制信息。支持用户通过用户代理（如浏览器）或网络接口使用网络。

TCP/IP参考模型

先有协议，后总结了模型。其共有四层：

  • 应用层：http…
  • 运输层：TCP,UDP
  • 网际层：ip
  • 网络接口层

特点：everything on ip 。

五层参考模型：

  • 应用层：http…
  • 传输层：进程-进程
  • 网络层：ip协议，路由协议
  • 数据链路层：相邻网络元素（主机、交换机、路由器等），以太网、wifi、PPP
  • 物理层：比特传输
